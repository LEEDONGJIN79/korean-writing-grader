# Workflow 5: 토론·합의 (Debate & Consensus) — v0.2

## 목적
1라운드 판정 결과를 보고 **편향·동조·과한 엄격성을 교정**한다. v0.2에서는 1라운드부터 해석형 항목에 보수+탐색 페르소나가 동시 의견을 내므로, 토론 라운드의 부담이 줄고 동조 위험도 감소한다.

## v0.2 핵심 변경 사항

- ✅ **양방향 자기 점검**: "반대 가설 검토"가 아니라 "찬·반 단서를 모두 재점검"
- ✅ **확신도 자연 하락 규칙**: 입장 변경 시 새 확신도 ≤ 기존 × 0.7
- ✅ **회색지대 디폴트**: 평가자가 망설일 때 변경이 아닌 회색지대로 전환
- ✅ **악마의 변호인 발동 조건 강화**: 1라운드 보수+탐색 다 미충족인 경우에만 발동

## 입력
- `outputs/[평가명]/per_student/[학번]_이름/01_round1.yaml`

## 출력
- `outputs/[평가명]/per_student/[학번]_이름/02_debate.yaml`
- `outputs/[평가명]/per_student/[학번]_이름/transcript_raw.md`

## Claude 처리 절차

### 단계 A: 1라운드 동조 진단 (메타 평가자)

1라운드 결과를 보고 다음 신호를 점검:

1. **해석형 항목의 보수·탐색 평가자가 같은 결론에 도달했는가** (만장일치 신호)
   - 보수+탐색이 모두 충족 → 신뢰할 만한 충족
   - 보수+탐색이 모두 미충족 → 동조 위험, 정밀 검토 필요
   - 보수와 탐색이 다른 결론 → 자연스러운 회색지대 신호

2. **객관형 평가자가 회색지대를 표시했는가**
   - 객관형은 보통 확신도 80%↑ → 회색지대 표시되면 해석형 재분류 검토

3. **같은 근거가 여러 평가자에게 반복 등장했는가** (예: "에세이-시 소재 불일치"가 두 항목에서 동시에)
   - 동조 신호로 보고

### 단계 B: 🆕 양방향 자기 점검 (모든 해석형 평가자)

각 해석형 평가자가 자기 1라운드 판정을 다음 방식으로 재점검한다.

#### "반대 가설 검토" → "자기 단서 재점검"으로 표현 변경

❌ 기존 (v0.1):
> "당신은 반대 가설을 적극 시도해야 합니다."

✅ v0.2:
> "당신의 1라운드 판정 후, 다음 두 가지를 한 번씩 점검하세요. 어느 쪽으로도 결론을 미리 정하지 마세요.
>
> **점검 1**: 당신의 판정을 흔들 만한 단서가 학생 자료에 있는가?
> **점검 2**: 당신의 판정을 뒷받침할 추가 단서가 학생 자료에 있는가?
>
> 두 점검 결과를 비교해 다음 중 하나를 선택하세요:
> - A. 기존 판정 유지 (이것이 정당한 디폴트입니다)
> - B. 회색지대(?)로 전환 (양쪽 단서가 비슷할 때)
> - C. 판정 변경 (학생 자료의 직접 단서로 뒷받침될 때만)"

### 단계 C: 🆕 확신도 자연 하락 규칙

입장이 바뀐 평가자는 **확신도가 자연스럽게 낮아져야 한다.** 변경 후에도 확신도가 그대로 높으면 동조 신호.

#### 규칙

```
입장 변경 시 새 확신도 ≤ 기존 확신도 × 0.7

예시:
- 1라운드: 미충족, 확신도 90%
- 재점검 후 충족으로 변경: 새 확신도 최대 63% (= 90 × 0.7)
- 90%를 유지한 채 충족으로 변경 → 동조 의심, 강제 회색지대로

기존 판정 유지 시: 확신도 유지 또는 미세 조정 자유
```

#### 평가자 prompt에 명시

> "당신이 입장을 변경한다면, 새 확신도는 기존 확신도의 0.7배 이하여야 합니다. 그 이상의 확신도로 변경할 수 없습니다. 만약 새 확신도가 0.7배 이상이라면, 변경 대신 회색지대(?)로 전환하세요."

### 단계 D: 🆕 회색지대 디폴트

평가자가 입장 변경을 망설일 때 **디폴트는 변경이 아닌 회색지대**다.

```
재점검 후 선택지의 무게:
  A. 기존 판정 유지 (가장 정당)
  B. 회색지대(?) (망설일 때의 디폴트)
  C. 판정 변경 (직접 단서가 명백할 때만)
```

평가자가 C를 선택하려면 **학생 자료의 직접 인용**으로 근거를 명시해야 한다. 인용 없는 변경은 강제로 B(회색지대)로 강등.

### 단계 E: 관대성 보정자 (해석형 항목 전반)

`prompts/charitable_reader.md` 페르소나로 `Agent` 도구 호출.

v0.2에서는 **1라운드부터 호의파 페르소나가 이미 있을 수 있으므로**, 관대성 보정자는 다음 경우에만 발동:
- 해석형 항목에 호의파가 배정되지 않은 경우
- 또는 호의파가 보수적 판정에 가까운 결과를 낸 경우

### 단계 F: 악마의 변호인 (조건 강화)

다음 조건이 모두 만족될 때만 발동:

```
1. 한 조건에 대해 보수 페르소나 + 탐색 페르소나 모두 미충족 판정
2. 양쪽 평가자의 확신도 평균 70% 이상
3. 메타 평가자가 동조 신호를 명시적으로 보고
```

이 조건이 충족되면 `prompts/devils_advocate.md` 페르소나 호출. **단 v0.2의 devils_advocate는 표현이 약화됨** (가설 제시자, 권고자 아님).

악마의 변호인 의견을 받은 평가자는 단계 B의 양방향 점검 + 단계 C의 확신도 자연 하락 규칙을 따른다.

### 단계 G: 메타 평가자 — 편향 자가 진단

`prompts/meta_evaluator.md` 페르소나로 호출.

v0.2에서는 **1라운드 동조 + 재토론 동조** 모두 점검:
- 1라운드: 보수+탐색이 같은 방향으로 갔는가?
- 재토론: 평가자들이 일제히 어느 한 방향(악마의 변호인 또는 관대성 보정자)으로 움직였는가?

### 단계 H: 결과 저장

각 조건의 최종 판정 (충족/미충족/회색지대) + 최종 확신도 + 모든 발언 누적을 `transcript_raw.md`에 시간순으로.

## 조건별 토론 정책 매트릭스 (v0.2)

| 1라운드 결과 | 다음 단계 |
|---|---|
| 객관형 + 충족 + 확신도 80%↑ | 토론 없음. 통과 |
| 객관형 + 미충족 + 확신도 80%↑ | 토론 없음. 통과 |
| 객관형 + 회색지대 | 해석형 재분류 → 양방향 자기점검 |
| 해석형 + 보수·탐색 모두 충족 (만장일치) | 양방향 자기점검만, 악마의 변호인 X |
| 해석형 + 보수·탐색 다른 결론 | 그것이 곧 회색지대. 추가 토론 없이 양쪽 의견 의장에 전달 |
| **해석형 + 보수·탐색 모두 미충족 (만장일치)** | **악마의 변호인 발동 + 양방향 자기점검** |

## 발언 예시 (양방향 자기점검 후)

### 예시 1: 기존 판정 유지

> "재점검 결과 — 단서를 다시 살펴봤습니다. 학생 자료에서 제 판정을 흔들 만한 새 단서를 발견하지 못했어요. 오히려 학생이 인용한 에세이 부분이 실제 본문에 없다는 점은 제 미충족 판정을 뒷받침합니다. 기존 판정 유지합니다. 확신도 85% 그대로."

### 예시 2: 회색지대로 전환

> "재점검 결과 — 양쪽 단서가 비슷합니다. 학생이 표면적으로 어긋난 글을 썼지만, 다층 해석 가능성도 부정하기 어렵습니다. 어느 쪽으로도 단정할 수 없어요. 회색지대(?)로 전환합니다. 학생 면담을 통한 의도 확인이 정직한 다음 단계입니다. 확신도 50%."

### 예시 3: 판정 변경 (확신도 자연 하락 적용)

> "재점검 결과 — 처음에 놓쳤던 단서가 있었습니다. 학생의 에세이 본문에 '13살 차이'라는 표현이 명시되어 있고, 이게 해설서의 '주변 사람'과 매핑 가능합니다. 직접 단서가 있으므로 충족으로 변경합니다. 다만 학생이 해설서에 그 매핑을 명시적으로 풀어내지 않은 점이 약점이라, 확신도는 65%로 자연 하락합니다 (기존 90% × 0.72)."

## 주의사항

- **양방향 자기점검은 결론 방향성을 암시하지 않는다.** "반대로 가라"가 아니라 "단서를 더 보라"는 메시지.
- **확신도 자연 하락은 강제 규칙이다.** 입장 변경 후 확신도가 0.7배 이상이면 시스템이 자동으로 회색지대로 강등.
- **회색지대는 시스템의 약점이 아니라 정직함이다.** 회색지대 비율이 높다고 시스템 실패가 아님.
- **악마의 변호인 발동이 줄어드는 것이 v0.2의 목표 중 하나.** 1라운드에 보수+탐색이 동시에 있으므로 동조 자체가 적게 일어남.
- 모든 단계의 발언을 `transcript_raw.md`에 누적 기록. 입장 변경 시 변경 이유와 새 확신도 명시.
