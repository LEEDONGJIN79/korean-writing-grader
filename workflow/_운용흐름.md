# 학급 단위 운용 흐름 — v0.8

## v0.8 핵심 변경: 외부 검증 파일 구조 단순화

v0.7까지는 외부 검증 자료가 각 학생 폴더(`per_student/[학번]_이름/external_check/`)에 흩어져 있었음. 교사가 학생마다 폴더를 들어가 검토 요청문·학생 자료를 합쳐 외부 모델에 던지는 번거로움.

v0.8 변경:
1. **모든 외부 검증 자료를 단일 폴더에 모음**: `outputs/[평가명]/_외부검증/`
2. **통합 검토 요청문 1개 파일에 모든 정보**: 과제명·성취기준·루브릭·학생 글·다른 평가자 의견 한꺼번에
3. **학생 자료 별도 첨부 불필요**: 검토 요청문 안에 이미 포함

학생당 작업 시간 약 5분 → **3~4분** (외부 모델 던지기·답변 받기·결과 입력만)

## v0.7 누적: "1명 검증 → 모드 선택 → 자동 완주"

v0.6까지는 학생 묶음마다 사용자가 응답을 기다려야 했음. 학급 30명이면 응답 10번 이상 사이클, 교사가 컴퓨터 앞에 묶임.

v0.7 변경:
1. **첫 학생 1명만 정밀 채점 → 사용자 검증** (시스템이 제대로 작동하는지 확인)
2. **사용자가 정석/효율 모드 선택**
3. **효율 모드**: 5명씩 묶어 사용자 확인 없이 끝까지 자동 진행 → 산출물 일괄 생성까지

→ **교사가 수업·회의·퇴근 후 돌아왔을 때 모든 결과가 완성**되어 있음.

### 정석 모드 vs 효율 모드

| | 정석 모드 | 효율 모드 |
|---|---|---|
| 묶음 크기 | 1~2명 | 5명 |
| 사용자 확인 | 묶음마다 | 처음 1명 후 한 번만 |
| 자리 비울 수 있는지 | 안 됨 | **가능 (전체 완주)** |
| 예상 응답 사이클 | N/2회 | 1회 (전체) |
| 적합한 상황 | 자세히 검토하며 진행 | 수업·회의 가야 할 때 |

자세한 행동 규칙은 `workflow/4_independent_judge.md` 참조.

## v0.6 누적: 외부 검증도 차등화

v0.5에서 산출물 생성을 차등화한 것에 이어, v0.6에서는 **외부 검증(3단계)도 🔴 필수검토 학생에 한정**.

| 라벨 | 산출물 | 외부 검증 |
|:---:|---|:---:|
| 🟢 자동확정 | 채점표 한 줄 | X |
| 🟡 권장검토 | 요약 (Claude 페르소나 결과로 충분) | **X** (v0.6 변경) |
| 🔴 필수검토 | 풀 산출물 | ✅ ChatGPT/Gemini |

학급 30명 기준 외부 검증 작업: 회색지대 5~8건 → **🔴 필수검토 3~4건** (시간 약 50% 추가 절감).

또한 v0.5까지 누락 가능성이 있던 **external_check/ 폴더 생성을 workflow/7에 명시적으로 박아 넣음**. 7단계에서 🔴 학생이 있으면 반드시 외부 검증 요청문을 자동 생성.

## v0.5 누적 변경: 차등 산출물 생성

기존 v0.4는 모든 학생에게 동일하게 풀 산출물(리포트·속기록)을 생성해 학급 30명 채점에 시간이 지나치게 오래 걸렸다. v0.5부터는 **검토 라벨에 따라 산출물 차등 생성**:

- 🟢 자동확정 (만장일치 충족): 산출물 X, 채점표 한 줄만
- 🟡 권장검토 (회색지대 1건): 요약 산출물 (1/4 분량)
- 🔴 필수검토 (회색지대 다수): 풀 산출물 (기존 수준)

학급 30명 기준 산출물 생성 시간이 약 **240분 → 74분** (70% 절감) 예상.

자세한 차등 정책은 `workflow/7_outputs.md` 참조.

---

이 문서는 **한 학급(20~30명) 단위로 채점을 진행할 때**의 권장 운용 흐름입니다. 워크플로우 단계(`1_setup` ~ `7_outputs`)는 학생 한 명을 채점하는 절차이고, 이 문서는 그 단계들을 **학급 규모에서 어떻게 묶어 운영할지** 설명합니다.

## 🎯 운용 5단계

```
1단계: 채점 완료 (🆕 v0.7 변경)

  ┌─ [1-A] 첫 학생 1명 정밀 채점 → 사용자 검증
  │       · 결과 보고 + "맞는지 확인해주세요"
  │       · 사용자: "계속해줘"
  │
  ├─ [1-B] 모드 선택 안내
  │       · 정석 모드: 묶음마다 결과 보고
  │       · 효율 모드: 5명씩 묶음, 자동 완주
  │       · 사용자 선택
  │
  └─ [1-C] 선택 모드로 나머지 학생 채점
          · 정석: 사용자 확인 따라 진행
          · 효율: 5명 묶음 자동 진행, 응답 한 번에 완주
          · 회색지대 학생·조건을 누적 큐로 기록
          · 결과는 yaml에만 (학생용 산출물은 2단계에서)
       ↓
2단계: 잠정 산출물 일괄 생성 (🆕 v0.5 차등 정책)
       └── 🟢 자동확정 학생: 채점표 한 줄 코멘트만 (리포트·속기록 X)
       └── 🟡 권장검토 학생: 요약 리포트·속기록 생성
       └── 🔴 필수검토 학생: 풀 리포트·속기록 생성
       └── 🔴 필수검토 학생의 `_외부검증/` 통합 폴더 일괄 생성 (v0.8)
       │    · 통합 검토 요청문 자동 생성 (과제·루브릭·학생글·다른 평가자 의견 모두 포함)
       │    · 결과 입력란 빈 양식 준비
       │    · README.md, _검토목록.md 자동 생성
       └── 잠정 채점표.md·csv·_채점요약.md 생성
       ↓
3단계: 🔴 필수검토 학생만 외부 검증 (교사 작업, v0.6 변경)
       └── 🔴 필수검토 학생만 외부 검증 대상
       │    · 🟡 권장검토는 외부 검증 X (요약 산출물 그대로 활용)
       │    · 🟢 자동확정은 당연히 X
       └── 교사가 outputs/[평가명]/_외부검증/ 폴더 열기 (v0.8: 모든 검토 요청문 한 곳에)
       └── 각 🔴 학생·조건에 대해:
       │    · 검토 요청문을 ChatGPT/Gemini에 던지기
       │    · 결과를 결과_입력란.md에 붙여넣기
       └── 모든 입력 완료 후 "외부 검토 완료" 알림
       └── ⏱️ 학급 30명 기준 🔴 필수검토 3~4건 → 약 15~20분 예상
       ↓
4단계: 산출물 자동 수정
       └── 외부 모델 결과 자동 파싱 (workflow/5_5)
       └── 의장이 세 시각 합의 판정 (Case 1~4 자동 분류)
       └── 영향받는 학생들의 리포트·속기록 자동 수정
       │    · "교사 확인 필요(?)" → 확정 표기로 변경
       │    · 외부 검증 라운드를 속기록에 추가
       └── 채점표·요약 보고서 업데이트
       ↓
5단계: 완료
       └── 최종 채점표.md·csv·_채점요약.md 확정
       └── 교사 최종 검토 권장 학생 목록 안내
       │    · 🟢 자동/외부검증 확정 (대부분)
       │    · 🟡 권장 검토 (확신도 60~80%)
       │    · 🔴 필수 검토 (외부 검증으로도 안 풀린 케이스)
       └── 교사가 최종 점수 확정 후 NEIS·기록부 입력
```

## 🔑 이 흐름의 이점

### 효율성
- **외부 모델 검증을 한 번에 묶음**: 회색지대 5건이라면 ChatGPT/Gemini를 한 번씩 열어 5건을 연속 처리. 학생별로 따로 하는 것보다 훨씬 빠름.
- **잠정 산출물을 일찍 확인**: 외부 검증 전이라도 교사가 전체 채점 결과를 미리 훑어볼 수 있음.

### 안전성
- **회색지대 학생만 외부 검증** → 비용 효율
- **검증 후 자동 수정**으로 휴먼 에러 감소
- **모든 변경 사항이 속기록에 누적** → 감사 추적 보장

### 실용성
- **교사 1회 작업 묶음**: 외부 검증을 한 번 앉아서 끝낼 수 있음 (학교 일과 중 30분 정도)
- **중간 산출물 확인 가능**: 회색지대 외 학생들 결과는 잠정 산출물에서 바로 확인

## 🛠️ Claude가 따라야 할 절차

### 1단계 (채점 완료) 명령
```
사용자: "[과제명]을 채점해줘. 루브릭은 [경로], 학생 자료는 [경로]"

Claude:
  1. workflow/1_setup.md ~ workflow/5_debate.md 를 모든 학생에 대해 실행
  2. 회색지대 학생·조건을 outputs/[평가명]/_회색지대_큐.yaml에 누적
  3. yaml 파일들(01_round1, 02_debate, transcript_raw)만 생성
  4. 학생용 산출물(리포트.md, 속기록.md)은 아직 X
  5. 사용자에게 "채점 완료. 잠정 산출물 생성으로 진행할까요?" 보고
```

### 2단계 (잠정 산출물) 명령
```
사용자: "잠정 산출물 만들어줘" 또는 1단계 후 자동 진행 동의

Claude:
  1. workflow/7_outputs.md를 전체 학생에 적용 (잠정 모드)
  2. 🔴 필수검토 학생의 `_외부검증/` 통합 폴더 자동 생성 (v0.8)
  3. 통합 검토 요청문 일괄 생성 (학생 글·루브릭·다른 평가자 의견 모두 포함)
  4. _검토목록.md 자동 생성 (교사가 어느 학생을 ChatGPT/Gemini에 보낼지 한눈에)
  5. 사용자에게 외부 검증 안내 + 회색지대 명단 보고
```

### 3단계 (외부 검증) 사용자 작업
교사가 `_회색지대_검토_안내.md`를 보고 ChatGPT/Gemini에 검토 요청문을 던집니다. 결과를 각 학생의 `결과_입력란.md`에 붙여넣은 후 "외부 검토 완료"라고 Claude에게 알립니다.

### 4단계 (자동 수정) 명령
```
사용자: "외부 검토 완료"

Claude:
  1. 각 회색지대 학생의 결과_입력란.md 파싱
  2. workflow/5_5_external_model_check.md의 합의 판정 로직 적용 (Case 1~4)
  3. 영향받는 학생들의 02_5_external.yaml, 03_chairman.yaml 업데이트
  4. 리포트.md·속기록.md 자동 수정
  5. 채점표.md·csv·_채점요약.md 업데이트
  6. 사용자에게 완료 보고 + 최종 검토 권장 학생 명단
```

### 5단계 (완료)
교사가 최종 검토 권장 학생을 직접 검토한 후, 채점표.csv의 "교사확정점수" 칸을 채워 NEIS 입력으로 마무리.

## ⚠️ 주의사항

- **2단계의 산출물은 "잠정"임을 명확히 표시.** 회색지대 학생의 리포트에는 "선생님과 함께 한 번 더 확인할 부분이 있어요" 같은 안내가 들어가야 함.
- **3단계 외부 검증을 건너뛰는 옵션도 허용.** 교사가 "외부 검증 생략"이라 하면 모든 회색지대가 🔴 필수검토로 라벨링되고 교사 직접 검토로 넘어감.
- **4단계 자동 수정은 회색지대 학생만 영향.** 다른 학생들의 산출물은 그대로 유지.
- **5단계 완료 후에도 교사가 점수를 변경할 수 있다.** 시스템은 보조 도구이지 최종 판정자가 아님.

## 단일 학생 채점 시 (운용 예외)

학생 1명만 빠르게 채점하고 싶을 때는 운용 흐름을 거치지 않고 워크플로우 1~7을 한 번에 진행해도 됩니다. 외부 검증이 필요하면 그 학생에 한해 5.5단계가 자동 발동됩니다.
