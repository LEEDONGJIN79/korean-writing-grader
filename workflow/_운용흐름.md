# 학급 단위 운용 흐름 — v0.9 (전면 재설계)

## v0.9 핵심 — 실제 학교 평가 절차 모방

v0.2~v0.8까지는 "평가자 4~6명 병렬 토론" 모델이었습니다. v0.9는 **실제 학교의 채점·이의제기·재심·최종 결정 절차**를 그대로 옮긴 모델입니다.

```
[학생 글]
   ↓
① 채점 교사 1명 (사용자가 엄격/느슨 선택)
   · 1차 채점 + 망설임 표시
   ↓
② 학생 (모든 학생, 고2 톤)
   · 점수 확인 → 동의하거나 이의제기
   ↓ (이의 있는 조건만)
③ 검토 교사 2명 (보수+탐색, 병렬)
   · 학생 이의 재심
   ↓
④ 의장 (사용자가 엄격/균형/느슨 선택)
   · 모든 의견 종합·최종 결정
   ↓ (의견 불일치 큰 경우만)
🔴 외부 모델 검토 요청 (ChatGPT/Gemini)
```

## 사용자 입력 단계 (시스템 시작 시)

```
[채점 시작 준비]

1. 과제명·성취기준·루브릭·학생 자료 폴더 (기존)

2. 🆕 채점 교사 수준
   🔵 엄격 — 글에 명백한 증거가 있어야 점수 인정
   🟢 느슨 — 학생의 시도를 적극 인정

3. 🆕 의장 수준
   🔵 엄격 — 망설임 → 낮은 쪽, 학생 이의 신중
   ⚖️ 균형 — 학생 이의·검토교사 의견 종합 (기본 권장)
   🟢 느슨 — 망설임 → 높은 쪽, 학생 시도 적극 인정

→ 선택 후 자동 채점 시작
```

## 운용 단계

```
[1단계] 자동 채점 (사용자 자리 비워도 됨)
  · 모든 학생에 대해 4단계 절차 진행
  · 의견 불일치 큰 경우 🔴 자동 표시
  ↓
[2단계] 산출물 생성 (자동)
  · 학생별 간단 리포트:
    - 잘한 점 (근거 인용)
    - 감점 사유 (근거 인용)
    - 학생 이의 (있는 경우) + 검토교사 의견 + 의장 결정
  · 채점표 (전체 점수)
  · 🔴 학생만 _외부검증/ 폴더에 검토 요청문 자동 생성
  ↓
[3단계] 외부 검증 (교사 작업, 🔴 학생만 약 15~20분)
  · ChatGPT/Gemini에 검토 요청
  · 결과 입력
  ↓
[4단계] 산출물 자동 수정 (외부 검증 결과 반영)
  ↓
[5단계] 완료
  · 최종 채점표
  · 교사 검토 후 NEIS 입력
```

## 산출물 양식 — 간단하게

각 학생의 리포트는 **A4 절반 분량** 정도로 간단:

```markdown
# [학번] [이름] 채점 결과

**총점**: 16점 / 20점
**채점 교사 수준**: 엄격
**의장 수준**: 균형

## 잘한 점
- **조건 ①** (5점): 시간·장소·인물 구체적
  > "중3 늦가을, 버스 정류장 5분이 30분처럼"
- **조건 ②** (4점): 자기 인식 또렷
  > "눈물의 질이 다르다"

## 감점 사유
- **조건 ③** (3점): 어법 오류·구어체 많음
  > "됬다" → "됐다" 등
- **조건 ④** (4점): 형 내면 추측이 짧음

## 학생 이의 (있는 경우)
> "조건 ②가 4점은 부당해요. 제가 행동 변화로 깨달음을 회수했거든요…"

**검토 결과**:
- 검토교사 보수: 부분 수용 (글 안 단서 약함)
- 검토교사 탐색: 부분 수용 (자기 인식이 행동의 시작점)
- **의장 결정**: 4점 유지 (균형 의장, 검토교사 종합)

🟡 권장검토 — 의견 일부 차이 있어 선생님 한 번 더 보시기 권장
```

🔴 외부 검토 학생의 경우 마지막 줄이:
> 🔴 의견 불일치 큼 — 외부 모델(ChatGPT/Gemini) 검토 요청 중. 결과 후 점수 확정.

## 차등화 적용 (v0.5~v0.6 기능 유지)

- 🟢 자동확정 (망설임 0건, 이의 없음): 간단 리포트만, 학생 이의 섹션 생략
- 🟡 권장검토 (망설임 1건, 학생 이의 일부 수용): 간단 리포트 + 이의 섹션
- 🔴 필수검토 (의견 불일치 큼): 외부 검증 요청

## 페르소나 호출 효율

| 단계 | 페르소나 | 학생당 호출 수 | 학급 30명 |
|---|---|:---:|:---:|
| 1단계 | 채점 교사 (선택된 1종) | 1 | 30 |
| 2단계 | 학생 | 1 (전원) | 30 |
| 3단계 | 검토교사 보수·탐색 (병렬) | 이의 시 1 회 (약 50%) | ~15 |
| 4단계 | 의장 (선택된 1종) | 1 | 30 |
| **합계** | | ~3.5 | **~105** |

학급 30명에 약 105 호출. v0.8과 비슷한 효율이면서 **흐름의 자연스러움과 깊이가 큼**.

## 정석 모드 vs 효율 모드 (v0.7 유지)

- 정석 모드: 묶음마다 결과 보고
- 효율 모드: 5명씩 묶어 자동 진행, 사용자 자리 비워도 됨

v0.9도 첫 학생 검증 + 모드 선택 흐름 유지.

## 이전 버전과의 차이 요약

| | v0.8 (이전) | v0.9 (현재) |
|---|---|---|
| 페르소나 모델 | 평가 철학 4~6명 병렬 | **역할 4단계 직렬 (8개 페르소나 중 선택)** |
| 학생 시각 | 없음 | **학생 페르소나로 도입** |
| 사용자 선택 | (없음) | **채점 교사 수준 + 의장 수준** |
| 회색지대 발견 | 페르소나 의견 갈림 | **학생 이의 + 검토교사 의견 차이** |
| 산출물 | 차등 (자동확정/요약/풀) | **간단 리포트 일관 + 이의 섹션** |
| 외부 검증 | 회색지대 다수 | **의견 불일치 큰 경우만** |
