# Workflow 2: 루브릭 파싱 (Parse Rubric)

## 목적
자유 형식(한글·Word·PDF·이미지·텍스트)으로 받은 루브릭을 시스템이 이해할 수 있는 구조로 변환하고, 그 결과를 사용자에게 확인받는다.

## 입력
- 루브릭 파일 경로

## 출력 (이 단계의 결과)
```yaml
과제명: (Workflow 1에서 받은 값)
배점: (전체 점수)
항목:
  - id: 1
    내용: (조건/평가 항목 본문)
    배점: (해당 항목 점수, 있으면)
    성격: (객관형/해석형 — 다음 단계에서 결정. 이 단계에서는 비워둠)
  - id: 2
    ...
점수표:
  (있다면 충족 개수 → 최종 점수 매핑)
  예: 5개_충족: 7, 4개_충족: 6, ...
```

## Claude 처리 절차

### 1. 파일 형식 자동 감지

```bash
file "[루브릭 파일 경로]"
```

형식별 처리 방법:
| 형식 | 처리 방법 |
|---|---|
| `.txt`, `.md` | Read 도구로 직접 읽기 |
| `.docx` | `textutil -convert txt -encoding UTF-8 "[file]" -output "[temp].txt"` 후 Read |
| `.hwp`, `.hwpx` | macOS 한글 변환기 또는 사용자에게 텍스트 추출 요청 |
| `.pdf` | Read 도구 (PDF 지원) 또는 `anthropic-skills:pdf` 스킬 |
| 이미지(`.png`, `.jpg`) | Read 도구로 직접 읽기 (Claude의 이미지 인식 활용) |

### 2. 항목 추출
파싱된 루브릭 본문에서 다음을 식별:
- **평가 항목/조건 목록** (보통 번호나 ①②③ 형태로 구분)
- **각 항목의 충족 기준** (있다면)
- **점수 산정 방식** (만점, 충족 개수→점수 매핑 등)
- **기본 점수**, **참여 점수** 등 특수 규정

### 3. 구조화 출력
위 YAML 형식으로 정리한다.

### 4. ⚠️ 사용자 확인 (필수)
파싱 결과를 사용자에게 보여주고 확인을 받는다. 임의로 다음 단계로 넘어가지 말 것.

```
[루브릭 파싱 결과]

과제명: 시 창작 및 자기 해설서 (7점)

평가 항목 5개:
  ① 시가 일정한 분량과 형식(행·연 구분)을 갖추어 작품으로서의 완성도가 있다.
  ② 감정경험 에세이에서 다룬 감정/경험을 바탕으로 시를 창작하고, 
     해설서에서 왜 이 감정/경험을 시로 썼는지 창작 의도를 설명하였다.
  ③ 시적 형식(비유, 이미지, 상징 등)을 의도적으로 활용하였고,
     해설서에서 핵심 표현의 선택 이유를 설명하였다.
  ④ 해설서에서 시의 구성(연 구분, 시상의 흐름 등)을 설명하였다.
  ⑤ 해설서에서 감정경험 에세이의 어떤 부분이 시의 어떤 부분에 반영되었는지
     구체적으로 연결하여 설명하였다.

점수 산정:
  · 5개 충족: 7점
  · 4개 충족: 6점
  · 3개 충족: 5점
  · 2개 충족: 4점
  · 1개 충족: 3점
  · 수업 참여만 (활동 미수행): 2점

이대로 진행할까요? 잘못 읽은 부분이 있으면 알려주세요.
```

### 5. 수정 요청 처리
사용자가 수정을 요청하면 해당 부분만 고친다. 다시 전체를 보여주고 재확인한다. 동의를 받을 때까지 반복한다.

### 6. 확정 후 저장
최종 파싱된 구조를 `outputs/[평가명]/_parsed_rubric.yaml` 에 저장한다 (감사 추적용).

## 주의사항

- **루브릭에 명시되지 않은 항목을 추가하지 말 것.** 시스템은 교사의 루브릭만 따른다.
- **점수표가 명시되지 않았으면 사용자에게 묻기.** 추측 금지.
- 항목 번호가 ①②③ 형태든 1·2·3이든 ⓐⓑⓒ든 일관되게 식별하되, 원문 표기는 보존한다.
- 이미지 루브릭은 OCR 오류 가능성이 있으므로 사용자 확인이 특히 중요하다.
