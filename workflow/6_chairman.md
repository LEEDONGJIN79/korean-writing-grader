# Workflow 6: 의장 최종 결정 (Chairman Decision) — v0.3

## 목적
모든 토론 결과를 종합해 학생별 최종 판정·점수·피드백 초안을 만든다. 회색 지대는 정직하게 "교사 확인 필요"로 표시하고, 성취기준 도달 정도를 진단한다.

## v0.3 변경 사항

- ✅ **외부 모델 의견 통합** (5.5단계 결과 반영)
- ✅ **세 의견 합의 패턴별 최종 판정 로직**
- ✅ **외부 모델로도 풀리지 않은 케이스는 🔴 교사 직접 검증 필수**

## 입력
- `outputs/[평가명]/per_student/[학번]_이름/02_debate.yaml`
- 🆕 `outputs/[평가명]/per_student/[학번]_이름/02_5_external.yaml` (회색지대 학생만)
- `outputs/[평가명]/per_student/[학번]_이름/transcript_raw.md`
- `outputs/[평가명]/_parsed_rubric.yaml`
- 성취기준

## 출력
- `outputs/[평가명]/per_student/[학번]_이름/03_chairman.yaml` (최종 판정)
- `outputs/[평가명]/per_student/[학번]_이름/chairman_remarks.md` (의장의 통합 코멘트)

## Claude 처리 절차

### 단계 1: 항목별 최종 판정 확정 (v0.3 업데이트)

토론 후 각 조건의 최종 상태를 정리:

| 상태 | 표기 | 점수 산정 시 |
|---|---|---|
| 만장일치 충족 + 확신도 평균 80%↑ | **충족 ✓** | 충족으로 계산 |
| 충족(재토론 후) + 확신도 60~80% | **충족 (재판정) ⚪** | 충족으로 계산 |
| 🆕 외부 모델 검증 합의 (Case 1·2) | **충족/미충족 ✓✗ (외부 검증)** | 그 결론으로 계산 |
| 회색지대 + 확신도 40~60% | **교사 확인 필요 ?** | 교사 판정 후 확정 |
| 🆕 외부 모델로도 풀리지 않은 회색지대 (Case 3) | **🔴 교사 직접 검증 필수 ??** | 교사 판정 필수 |
| 🆕 Claude 합의 vs 외부 모델 반대 (Case 4) | **🟡 이의제기 가능 (Claude로 잠정)** | 교사 검토 권장 |
| 미충족 + 확신도 60%↑ | **미충족 ✗** | 미충족으로 계산 |

### 🆕 단계 1.5: 외부 모델 결과 통합 (v0.3 신규)

만약 `02_5_external.yaml`이 존재하면 (해당 학생에 회색지대가 있었다면):

#### Case 1: 셋 다 같은 결론
```
Claude 보수 평가자: 미충족
Claude 탐색 평가자: 미충족
외부 모델: 미충족
→ 미충족으로 합의. 확신도 매우 높음 (셋이 독립적으로 같은 결론)
→ 표기: "미충족 ✗ (3개 시각 합의)"
```

#### Case 2: Claude 분열, 외부가 결정자
```
Claude 보수 평가자: 미충족
Claude 탐색 평가자: 충족
외부 모델: 충족 (확신도 70%+)
→ 외부 모델 방향으로 합의
→ 표기: "충족 ✓ (외부 검증 결정)"
→ 확신도: Claude 분열을 외부가 해소한 신뢰 (보통 60~70%)
```

#### Case 3: 세 의견 모두 다르거나 외부도 회색지대
```
Claude 보수 평가자: 미충족
Claude 탐색 평가자: 충족
외부 모델: 회색지대 또는 다른 결론
→ 🔴 교사 직접 검증 필수
→ 표기: "교사 검증 필수 ??"
→ 세 시각이 일치하지 않는 본질적 회색지대. 교사 면담 권장.
```

#### Case 4: Claude 합의 vs 외부 반대
```
Claude 보수 평가자: 충족
Claude 탐색 평가자: 충족
외부 모델: 미충족
→ Claude 합의가 우선이지만 외부 모델 이의 표시
→ 표기: "충족 (잠정) 🟡 이의제기 가능"
→ 교사가 외부 모델 의견을 보고 재고할 수 있도록 정보 제공
```

### 단계 2: 점수 산정

루브릭의 점수표를 그대로 따른다. 두 가지 점수를 보고:

- **잠정 점수**: 회색지대를 모두 "미충족"으로 가정한 최저 점수
- **상한 점수**: 회색지대를 모두 "충족"으로 가정한 최고 점수

```yaml
잠정_점수: 4  # 회색지대 미충족 가정
상한_점수: 6  # 회색지대 충족 가정
교사_확인_필요: 2건  # 조건 ①, ⑤
```

만약 회색지대가 없으면 잠정 = 상한 = 확정 점수.

### 단계 3: 강점·보완점 정리

토론 속기록에서 추출:
- **강점**: 충족 항목 중 신뢰도 높음 + 인용이 인상적인 부분 2~3개
- **보완점**: 미충족·회색지대 항목의 핵심 사유

각각 학생 자료의 **직접 인용**을 1구절씩 첨부한다.

### 단계 4: 성취기준 도달 진단

성취기준의 핵심 동사·키워드를 추출 (예: "이해", "감상", "평가", "생산").
루브릭 항목과 성취기준 키워드를 매핑한다.

```yaml
성취기준: "[12문학01-03] 문학과 인접 분야의 관계를 바탕으로 작품을 이해·감상하고 평가한다"
도달_진단:
  이해: 충족 (조건②, ⑤ 기반)
  감상: 충족 (조건③ 기반)
  평가: 회색지대 (조건④ 부분 도달)
종합: "성취기준에 충분히 도달. 작품과 자기 경험의 관계를 메타적으로 사유하는 능력이 또렷이 관찰됨."
```

성취기준의 동사가 루브릭 항목과 직접 연결되지 않으면, **솔직하게 "이 항목으로는 측정 불가"** 라고 표기한다.

### 단계 5: 의장 통합 코멘트 작성

학생 리포트의 "총평" 섹션이 될 글. 200~400자.

다음 구조:
1. **이 학생 답안의 가장 두드러진 특징** (1~2문장)
2. **인상적인 강점** (1문장 + 인용)
3. **다음 글쓰기에서 다듬을 점** (1문장)
4. **성취기준 도달에 대한 한 줄 진단**

⚠️ 절대 피해야 할 표현:
- "AI가 보기에는..."  → 학생을 평가하는 주체는 교사. AI는 보조.
- "기계적으로", "자동으로" → 학생에게 차가운 느낌
- "잘했다"의 의미 없는 칭찬 → 구체적 인용이 없으면 무의미

### 단계 6: 교사용 메모 (별도)

학생 리포트에는 들어가지 않는, **교사 검토용** 한 줄 요약:

```yaml
교사_메모:
  - "조건⑤ 회색지대 — 학생이 1:1 매핑을 명시적으로 적지 않음. 학생 면담 권장."
  - "에이전트들이 처음에 '연애시' 편향에 끌려간 흔적. 다층 해석 후 충족 재판정."
  - "전체 확신도 평균 75% — 보통 수준의 케이스."
```

### 단계 7: 검토 우선순위 라벨링 (v0.3 업데이트)

다음 라벨을 자동 부여 (엑셀 채점표에 사용):
- 🟢 **자동확정** — 모든 항목 확신도 80%↑, 회색지대 0
- 🟢 **외부검증 확정** — 회색지대가 외부 모델로 합의된 케이스 (Case 1·2)
- 🟡 **권장검토** — 확신도 평균 60~80% 또는 외부 모델 이의제기 (Case 4)
- 🔴 **필수검토** — 외부 모델로도 풀리지 않은 회색지대 (Case 3) 또는 회색지대 2건 이상

## 의장의 책무 요약

1. **점수를 단정하지 않는다.** 잠정·상한 두 점수와 회색지대를 명시.
2. **학생을 평가하는 사람의 자리는 교사**임을 잊지 않는다.
3. **성취기준에 맞춰 피드백을 풀어낸다.** 점수 설명이 아니라 성장 방향.
4. **토론 과정의 정직함을 학생 리포트로 옮긴다.** 만장일치였다가 재판정 변경된 경우, "처음엔 다르게 봤지만 다시 읽고 보니..."는 학생에게 보여줄 수 있는 학습 자료가 된다 (선생님 재량).

## 주의사항

- 강점·보완점의 인용은 **반드시 학생 작품 원문에서** 가져온다. 일반 진술 금지.
- 성취기준 도달 진단이 어려우면 "이 루브릭으로는 측정 한계가 있음"이라고 정직히 적는다.
- 의장은 토론 내용을 **요약**하는 사람이 아니라, **종합하여 책임 있는 보고서**를 만드는 사람이다.
