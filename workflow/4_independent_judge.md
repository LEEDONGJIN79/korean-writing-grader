# Workflow 4: 채점 4단계 절차 (Real-School Assessment) — v0.9

## v0.9 완전 재설계

v0.8까지의 "페르소나 4~6명 병렬 토론"을 폐기하고, **실제 학교 평가 절차**를 그대로 옮긴 4단계 직렬 모델로 전환.

```
[학생 글]
   ↓
① 채점 교사 1명 (사용자가 엄격/느슨 선택)
   ↓
② 학생 페르소나 (모든 학생, 고2 톤)
   ↓ (이의 있는 조건만)
③ 검토 교사 2명 (보수+탐색, 병렬)
   ↓
④ 의장 (사용자가 엄격/균형/느슨 선택)
   ↓
[최종 점수 + 간단 리포트]
   ↓ (의견 불일치 큰 경우만)
🔴 외부 모델 검토 (workflow/5_5)
```

## 입력
- 학생 산출물 (전문)
- 평가 정보 (과제명·성취기준·루브릭)
- 사용자 선택 (채점교사 수준·의장 수준)

## 4단계 절차

### 단계 ① 채점 교사 — 1차 채점

**호출 페르소나**: 사용자가 선택한 `채점교사_엄격.md` 또는 `채점교사_느슨.md`

**prompt 구조**:
```
[페르소나 prompt 본문]

[평가 정보]
- 과제명: ...
- 성취기준: ...
- 루브릭: ...

[학생 답안]
... 학생 글 전문 ...

[지시]
조건별로 점수를 부여하되, 망설이는 조건은 'X점 ~ Y점 망설임'으로 표시.
근거를 글의 직접 인용으로.
```

**출력**: 조건별 점수 + 망설임 표시 + 원_발언

**저장**: `outputs/[평가명]/per_student/[학번]_이름/01_채점교사.yaml`

---

### 단계 ② 학생 — 이의제기

**호출 페르소나**: `prompts/personas/학생.md`

**prompt 구조**:
```
[페르소나 prompt 본문 — 고2 학생 톤]

[당신이 받은 점수]
... 채점 교사의 1차 점수 + 망설임 표시 ...

[당신이 쓴 글]
... 학생 글 전문 ...

[지시]
이 점수에 대해 동의하거나 이의 제기.
확신하는 부분은 짧게 "이 점수 동의해요"
부당하다고 느끼면 적극 변호 (정직한 자기 옹호).
```

**출력**: 이의 여부 + 이의 조건별 변론 (있는 경우)

**저장**: `outputs/[평가명]/per_student/[학번]_이름/02_학생이의.yaml`

---

### 단계 ③ 검토 교사 — 학생 이의 재심 (이의 있을 때만)

학생 이의가 없으면 이 단계 **건너뜀**.

이의가 있으면 검토교사 2명을 **병렬 호출**:

**호출 페르소나**:
- `prompts/personas/검토교사_보수.md`
- `prompts/personas/검토교사_탐색.md`

**prompt 구조 (각 페르소나에 동일)**:
```
[페르소나 prompt 본문]

[채점 교사 1차 점수]
... 단계 ① 결과 ...

[학생 이의]
... 단계 ② 결과 ...

[학생 글] (참고)
... 학생 글 전문 ...

[지시]
학생 이의를 [보수/탐색] 시각에서 재심.
학생 주장이 글 안의 근거로 뒷받침되는지 판정.
수용 / 부분 수용 / 기각 중 선택.
```

**출력 (2명)**: 각 조건별 수용 여부 + 수정 점수 + 원_발언

**저장**: `outputs/[평가명]/per_student/[학번]_이름/03_검토교사.yaml` (보수+탐색 결과 통합)

---

### 단계 ④ 의장 — 최종 결정

**호출 페르소나**: 사용자가 선택한 `의장_엄격.md` / `의장_균형.md` / `의장_느슨.md`

**prompt 구조**:
```
[페르소나 prompt 본문 — 선택된 의장 수준]

[채점 교사 1차 점수]
... 단계 ① 결과 ...

[학생 이의] (있는 경우)
... 단계 ② 결과 ...

[검토 교사 의견] (있는 경우)
... 단계 ③ 결과 (보수+탐색) ...

[지시]
모든 의견 종합 → 최종 점수 결정.
의장 수준에 따라 망설임 처리:
  엄격: 망설임 → 낮은 쪽
  균형: 학생 이의·검토교사 종합
  느슨: 망설임 → 높은 쪽, 학생 시도 적극 인정

다음에 해당하면 🔴 외부 검토 트리거:
  - 검토교사 둘의 점수 차이 2점 이상
  - 학생 이의가 강한데 검토교사 의견 갈림
  - 채점교사·학생·검토교사 셋이 모두 다른 방향
```

**출력**: 조건별 최종 점수 + 총점 + 외부 검토 트리거 여부 + 의장 코멘트

**저장**: `outputs/[평가명]/per_student/[학번]_이름/04_의장.yaml`

## 학급 단위 처리 — 팀 수에 따른 분기 (v1.0)

사용자가 시작 시 선택한 **채점 팀 수**에 따라 처리 흐름이 달라진다.

### 옵션 A: 1팀 (순차)

```
[학생 1] ①→②→③→④
    ↓
[학생 2] ①→②→③→④
    ↓
...
[학생 N] ①→②→③→④
    ↓
[산출물 일괄 생성]
```

- 동시 호출 수: 1 (한 단계당 1 Agent)
- 학급 25명 기준 약 30분~1시간
- 가장 안전, 토큰 효율 최선
- 적은 인원 / 정밀 검증 / 초보 사용자 권장

### 옵션 B: N팀 (소규모/중간 병렬) — **추천 (N=5)**

```
[묶음 1: 학생 1~N] 동시
   단계 ①: N 학생 동시 채점
   단계 ②: N 학생 동시 학생이의
   단계 ③: 이의 학생만 동시 검토 (보수+탐색)
   단계 ④: N 학생 동시 의장
    ↓
[묶음 2: 학생 N+1~2N] 동시
   같은 절차
    ↓
...
[모든 묶음 끝]
    ↓
[산출물 일괄 생성]
```

- 동시 호출 수: 단계 ③에서 최대 2N (보수+탐색)
- 5팀이면 한 묶음에 5명, Agent 한도(25) 안에서 안전
- 학급 25명 기준 약 5~10분
- 가장 균형 잡힌 옵션, **일반 수행평가 권장**

### 옵션 C: 전체 (인원수만큼)

```
[전체 학생] 단계 ① 동시 채점
   ↓ 모두 끝나면
[전체 학생] 단계 ② 동시 학생이의
   ↓
[이의 학생] 단계 ③ 동시 검토 (보수+탐색)
   ↓
[전체 학생] 단계 ④ 동시 의장
   ↓
[산출물 일괄 생성]
```

- 동시 호출 수: 학생 수 (단계 ③ 시 최대 학생 수×2)
- 학생 25명 이하만 단일 호출 가능. 초과 시 자동 분할.
- 학급 25명 기준 약 2~3분 (가장 빠름)
- 토큰 비용은 살짝 높지만 시간 우선일 때 선택

### 첫 학생 정밀 채점 → 자동 완주 (v0.7 계승)

옵션 B·C 선택 시에도 권장:
```
[첫 학생 1명 정밀 채점 (4단계 완전 진행)]
   ↓ 결과 보고
[사용자: "맞아, 계속해줘"]
   ↓
[선택한 팀 수로 자동 완주]
   ↓
[산출물 일괄 생성]
   ↓
[🔴 외부 검토 학생 별도 폴더에 패키지 생성]
```

### 효율 모드 행동 규칙 (v0.7 계승)

1. 묶음 끝났다고 사용자 확인 묻지 말 것
2. 다음 묶음으로 즉시 진행
3. 한 학생당 단계 ①·②·④ + (이의 있으면 ③) 모두 실행
4. 결과 요약은 한 줄로
5. 오류 발생 시에만 멈춤
6. 모든 학생 완료 후 산출물 일괄 생성
7. 🔴 학생의 외부 검증 자료 자동 생성
8. 최종 보고는 한 번만

## 페르소나 호출 효율 (학생 30명 기준)

| 단계 | 페르소나 | 호출 수/학생 | 학급 30명 |
|---|---|:---:|:---:|
| ① 채점교사 | 1종 (선택) | 1 | 30 |
| ② 학생 | 1종 | 1 (전원) | 30 |
| ③ 검토교사 | 2종 병렬 (이의 시) | ~1 (이의 ~50%) | ~15 |
| ④ 의장 | 1종 (선택) | 1 | 30 |
| **합계** | | ~3.5 | **~105** |

## 산출물 통합

단계 ①②③④ 결과를 의장이 `03_chairman.yaml` 형식으로 통합하면, `workflow/7_outputs.md`가 그것을 학생용 리포트(잘한 점·감점 사유·이의·검토·결정)로 변환.

## 주의사항

- **모든 학생에 ②단계 학생 이의 단계를 적용**. 확신 케이스에서는 학생이 짧게 "동의해요"로 종료.
- **③단계는 이의 있을 때만**. 이의 없는 학생은 ①②④만.
- **외부 검토는 의장의 판단**. 의장이 트리거 조건 충족 학생만 _외부검증/ 폴더에 패키지 생성.
