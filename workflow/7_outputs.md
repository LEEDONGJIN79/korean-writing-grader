# Workflow 7: 산출물 생성 (Outputs) — v0.5

## 목적
의장의 최종 판정을 바탕으로 산출물을 생성하되, **검토 라벨에 따라 차등 생성**한다. 시간·토큰 효율을 위해 자동확정 학생은 최소 정보만, 필수검토 학생만 풀 산출물.

## ⭐ v0.5 핵심 변경: 차등 산출물 정책

| 라벨 | 학생 리포트 | 속기록 | 예상 토큰·시간 |
|:---:|---|---|:---:|
| 🟢 **자동확정** | 채점표 한 줄 코멘트만 (리포트.md 생성 X) | 생성 X (yaml만 보관) | **약 80% 절감** |
| 🟡 **권장검토** | **요약 리포트** (500~700자) | 요약 속기록 (회색지대 항목만) | 약 50% 절감 |
| 🔴 **필수검토** | 풀 리포트 (2000~3000자) | 전체 속기록 | 변함 없음 |

### 사용 템플릿
- 자동확정: 별도 파일 없음, `채점표.md`의 학생 행에 한 줄 코멘트만
- 권장검토: `templates/student_report_summary.md` + `templates/debate_transcript_summary.md`
- 필수검토: `templates/student_report.md` + `templates/debate_transcript.md` (기존)

### 라벨 결정 기준 (의장이 03_chairman.yaml에 명시)
- 🟢 자동확정: 모든 항목 확신도 80%↑ + 회색지대 0건 + 외부 검증 불필요
- 🟡 권장검토: 회색지대 1건 또는 확신도 평균 60~80% 또는 외부 검증 후 Case 1·2 합의
- 🔴 필수검토: 회색지대 2건 이상 또는 외부 검증으로도 안 풀린 Case 3

## 입력 (학생마다)
- `outputs/[평가명]/per_student/[학번]_이름/03_chairman.yaml`
- `outputs/[평가명]/per_student/[학번]_이름/chairman_remarks.md`
- `outputs/[평가명]/per_student/[학번]_이름/transcript_raw.md`

## 출력 (학생마다, 라벨에 따라 차등)

### 🟢 자동확정 학생
- (yaml만 보관, 별도 산출물 파일 생성 X)
- 채점표.md의 학생 행에 한 줄 코멘트만 추가

### 🟡 권장검토 학생
- `outputs/[평가명]/per_student/[학번]_이름/리포트.md` (**요약 버전**)
- `outputs/[평가명]/per_student/[학번]_이름/속기록.md` (**요약 버전**)

### 🔴 필수검토 학생
- `outputs/[평가명]/per_student/[학번]_이름/리포트.md` (**풀 버전**)
- `outputs/[평가명]/per_student/[학번]_이름/속기록.md` (**풀 버전**)

## 출력 (전체)
- `outputs/[평가명]/채점표.xlsx`
- `outputs/[평가명]/_채점요약.md` (전체 통계·검토 우선순위 학생 목록)

## Claude 처리 절차 — v0.5

### 단계 1: 라벨별 학생 분류
의장의 03_chairman.yaml에서 검토 라벨을 확인해 학생을 세 그룹으로 분류:
- 🟢 자동확정 학생 명단
- 🟡 권장검토 학생 명단
- 🔴 필수검토 학생 명단

### 단계 2: 🔴 필수검토 학생 풀 산출물 + 🆕 외부 검증 요청 패키지 생성 (v0.6)
교사가 가장 먼저 봐야 할 학생들. 두 가지를 함께:

**(2-1) 풀 산출물 생성:**
- `templates/student_report.md` 기반 풀 리포트
- `templates/debate_transcript.md` 기반 전체 속기록

**(2-2) 🆕 v0.8 통합 검토 요청문 생성 (별도 폴더에 모음):**

위치를 **학생 폴더가 아닌 별도 통합 폴더**로 변경:

```
outputs/[평가명]/_외부검증/                ← v0.8 신설
├── README.md                            (작업 안내, 자동 생성)
├── _검토목록.md                          (전체 명단·진행 상태, 자동 생성)
├── [학번]_[이름]_조건N.md                 통합 검토 요청문 (모든 정보 포함)
└── [학번]_[이름]_조건N_결과.md            결과 입력란
```

각 🔴 필수검토 학생의 회색지대 조건마다 **단일 통합 검토 요청문** 생성:
- `prompts/external_model_request.md` v0.8 템플릿 기반
- **하나의 파일에 모두 포함**: 과제명·성취기준·전체 루브릭·조건·학생 글 전문·다른 평가자 의견·함정 패턴 메모
- 교사는 이 파일 하나만 복사해 ChatGPT/Gemini에 던지면 됨 (다른 파일 합칠 필요 X)
- 학생 글 본문의 학번·이름은 자동 익명화 ([학번/이름 제거됨])

⚠️ **`per_student/[학번]_이름/external_check/` 폴더는 v0.8부터 사용하지 않음.** 모든 외부 검증 자료는 `_외부검증/` 통합 폴더에.

⚠️ **이 단계가 누락되면 외부 검증이 안 됨.** 반드시 실행할 것.

### 단계 3: 🟡 권장검토 학생 요약 산출물 생성 (외부 검증 X)
- `templates/student_report_summary.md` 기반 요약 리포트
- `templates/debate_transcript_summary.md` 기반 요약 속기록
- 회색지대 항목 중심으로 압축
- **`_외부검증/` 폴더에 포함되지 않음** (외부 검증 대상 아님, v0.8)

### 단계 4: 🟢 자동확정 학생 처리 (가장 빠름)
- 산출물 파일 **생성하지 않음**
- 채점표.md/csv에 학생 행 + 한 줄 코멘트만 추가
- yaml 파일(01_round1, 03_chairman)은 그대로 보관 (감사 추적)

### 단계 5: 외부 검증 안내 보고 (🔴 학생 있을 때, v0.8 갱신)
🔴 필수검토 학생이 있으면 사용자에게 안내:
```
[v0.8 외부 검증 단계]

🔴 필수검토 학생 X명에 대해 통합 검토 요청문을 생성했습니다.
모든 파일이 한 폴더에 모여 있습니다:

📂 outputs/[평가명]/_외부검증/
   ├── README.md          (작업 안내)
   ├── _검토목록.md         (전체 명단)
   ├── 2-3-2_강종민_조건②.md     ← 통합 검토 요청문
   ├── 2-3-2_강종민_조건②_결과.md  ← 답변 붙여넣을 자리
   ├── 2-3-5_김진호_조건④.md
   ├── 2-3-5_김진호_조건④_결과.md
   └── ...

선생님 작업:
1. _검토목록.md를 먼저 열어 전체 명단 확인
2. 각 학생 파일을 열어 전체 복사
3. ChatGPT/Gemini에 붙여넣기 → 답변 받기
4. 결과 파일에 붙여넣기 → 저장
5. 모든 완료 후 "외부 검토 완료" 알림

학생당 약 3~4분 예상 (학생 자료가 검토 요청문에 이미 포함됨).

🟡 권장검토 학생 Y명: 외부 검증 없이 요약 산출물 생성 완료
🟢 자동확정 학생 Z명: 채점표에 정리 완료
```

### 단계 6: 전체 채점표·요약 보고서 생성
- 채점표.md / 채점표.csv / _채점요약.md
- 모든 라벨 학생 포함 (점수·라벨·한 줄 코멘트)
- 외부 검증 진행 상황 표시 (🔴 학생만)

## 1️⃣ 풀 리포트 (🔴 필수검토만)

### 템플릿
`templates/student_report.md` 의 형식을 따른다.

### 작성 원칙
- **학생 공개용**이다. 학생의 자존감과 학습 의지를 해치지 말 것.
- 점수만 통보하지 말고 **성취기준 도달 정도**와 **다음 성장 방향**을 풍부히.
- 학생 작품의 **직접 인용**을 풍부하게 사용 (자기 글이 인용되는 경험 자체가 학습 효과).
- **속기록의 반론·악마의 변호인 발언은 포함하지 말 것** (교사 검토 전용).
- **회색지대가 있는 경우** "이 부분은 선생님과 함께 한 번 더 짚어보자" 같은 부드러운 안내.

### 점수 표기
- 회색지대 0건: 점수 단정 표기 ("7/7점")
- 회색지대 1건 이상: "선생님 확인 후 확정" + 잠정 점수 표기 ("잠정 6점, 선생님 확인 후 최대 7점")

### 길이 가이드
- A4 1~2장 분량 (1500~3000자)
- 너무 길면 학생이 끝까지 읽지 않음

---

## 2️⃣ 토론 속기록 생성

### 템플릿
`templates/debate_transcript.md` 의 형식을 따른다.

### 작성 원칙
- **`transcript_raw.md` 의 모든 발언을 시간 순서대로 그대로** 옮긴다.
- 요약·정리하지 말 것. 발언을 그대로 살릴 것 (이게 핵심 차별점).
- 토론 라운드 헤더만 추가 (【1라운드】, 【2라운드 — 악마의 변호인】 등).
- 시간 표기는 단순화 (예: "회의 시작 + 0:00", "+ 0:03" 등)

### 파일 머리에 명시
- 학생 정보
- 회의 일시
- 참석 에이전트 명단
- **"교사 검토 전용 — 학생에게 공개하지 마세요"** 경고 박스

### 길이
- 학생 1명당 1~5쪽 (학생 답안 복잡도에 따라 다름)

---

## 3️⃣ 채점표 엑셀 생성

### 사용 도구
`anthropic-skills:xlsx` 스킬을 호출해 생성한다.

### 시트 구성

#### 시트 1: 채점표 (메인)
| 학번 | 이름 | 조건①점수 | 조건②점수 | ... | 잠정점수 | 상한점수 | 검토라벨 |
|------|-----|---------|---------|-----|---------|---------|---------|
| 2-3-1 | 강건우 | O(98%) | O(95%) | ... | 7 | 7 | 🟢 자동확정 |
| 2-3-2 | 강종민 | O(70%) | O(60%) | ... | 5 | 7 | 🟡 권장검토 |
| 2-3-3 | 권민기 | ?(40%) | O(85%) | ... | 5 | 7 | 🔴 필수검토 |

- **확신도 60% 미만 셀은 노란색** 자동 배경
- **회색지대(?) 셀은 빨간색** 배경
- 셀에 마우스 올리면 인용·근거 코멘트 표시 (Excel 메모 기능)

#### 시트 2: 개인별 코멘트
학생마다 의장 통합 코멘트와 강점·보완점을 한 행씩.

#### 시트 3: 루브릭 사본
파싱된 루브릭 원문 + 항목 번호·내용·배점.

#### 시트 4: 성취기준
성취기준 코드 + 내용 + 도달 진단 통계 (전체 학생 중 몇 명이 도달했는지).

#### 시트 5: 검토 우선순위
🔴 필수검토 → 🟡 권장검토 → 🟢 자동확정 순으로 정렬된 학생 목록. 선생님이 이 시트만 보고 어디부터 검토할지 결정.

### 파일명
`[평가명]_채점표_[날짜].xlsx`
예: `시창작_해설서_채점표_2026-05-24.xlsx`

## 4️⃣ 전체 요약 보고서 (`_채점요약.md`)

선생님이 한눈에 보는 메타 보고서:

```markdown
# 채점 완료 — 시 창작 및 자기 해설서 (2-3반)
일시: 2026-05-24
대상: 26명

## 점수 분포
- 7점: 8명
- 6점: 10명
- 5점: 5명
- 4점 이하: 3명

## 검토 우선순위
🔴 필수검토 (3명): 강종민, 권민기, 양준혁
🟡 권장검토 (7명): ...
🟢 자동확정 (16명): ...

## 발견된 흥미로운 패턴
- 26명 중 5명이 "에세이-시 1:1 매핑"(조건⑤)에서 회색지대 — 
  과제 안내 시 이 부분을 더 강조할 필요
- 조건① 형식 미충족 학생들 공통점: 연 구분 빈 줄 없이 한 덩어리로 제출
- ...

## 다음 단계 권장
1. 🔴 3명 면담 또는 답안 재검토 후 점수 확정
2. 🟡 7명 코멘트 검토 (큰 수정 없을 가능성)
3. 🟢 16명 자동 통과 가능 (랜덤 3~5명 샘플 검증 권장)

## 시스템 자체 메모
- 메타 평가자가 "조건② 평가자가 연애시 편향에 끌려간 흔적" 1건 보고 → 강종민 케이스
- 다층 해석 후 점수 변경 사례: 2건
```

## 5️⃣ 폴더 구조 (최종)

```
outputs/
└── [평가명]/
    ├── _parsed_rubric.yaml
    ├── _agent_assignments.yaml
    ├── _채점요약.md
    ├── 채점표.xlsx
    │
    └── per_student/
        ├── 2-3-1_강건우/
        │   ├── 01_round1.yaml
        │   ├── 02_debate.yaml
        │   ├── 03_chairman.yaml
        │   ├── transcript_raw.md
        │   ├── 리포트.md           ← 학생 공개용
        │   └── 속기록.md           ← 교사 검토 전용
        ├── 2-3-2_강종민/
        └── ...
```

---

## Claude 처리 절차

1. 모든 학생 처리가 끝난 후 이 워크플로우 시작
2. 학생마다 리포트·속기록 생성 (순차 또는 배치)
3. 채점표 엑셀 생성 (`anthropic-skills:xlsx`)
4. 전체 요약 보고서 작성
5. 사용자에게 완료 보고 + 검토 우선순위 안내

```
[채점 완료]

생성된 파일:
· 학생 리포트: 26개 (outputs/시창작_해설서/per_student/*/리포트.md)
· 토론 속기록: 26개 (교사 검토 전용)
· 채점표 엑셀: outputs/시창작_해설서/채점표.xlsx
· 요약 보고서: outputs/시창작_해설서/_채점요약.md

⚠️ 우선 검토 권장:
🔴 필수검토 3명: 강종민, 권민기, 양준혁
   → 속기록 함께 보세요. 회색지대 또는 입장 변경이 있었습니다.

다음 행동:
1. _채점요약.md를 먼저 보시면 전체 그림이 잡힙니다
2. 🔴 학생 3명의 속기록을 우선 검토하세요
3. 채점표 엑셀에서 노란색/빨간색 셀이 우선 검토 대상입니다
```

## 주의사항

- 학생 리포트와 속기록을 **혼동해서 같은 폴더에 두지 말 것** — 우발적 공개 방지를 위해 폴더 내 구분 명확히.
- 엑셀 생성 실패 시 사용자에게 알리고 CSV 대안 제공.
- 학생 리포트에는 다른 학생의 점수·이름이 절대 들어가지 않도록 주의.
- 모든 산출물에 생성 일시·시스템 버전을 푸터에 기록.
