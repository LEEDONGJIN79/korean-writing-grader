# Workflow 5.5: 외부 모델 교차 검증 (External Model Check) — v0.6

## 목적
v0.2의 보수+탐색 페르소나 토론 후에도 합의가 어려운 **🔴 필수검토 학생**에 대해, **다른 LLM 모델**의 독립 의견을 받아 합의를 시도한다. 그래도 합의가 안 되면 교사 직접 검증으로 에스컬레이션한다.

## v0.6 핵심 변경 (이전 v0.3에서 강화)

같은 Claude 모델 안의 페르소나 다양성은 본질적 한계가 있다. 진짜 다른 학습 데이터·다른 모델(GPT, Gemini 등)의 의견을 더하면 **편향이 본질적으로 다른 차원**에서 보정된다.

**v0.3 → v0.6 변경 사유**: v0.3는 "회색지대 학생 전부"를 외부 검증으로 보냈으나, 학급 30명에서 회색지대가 5~8명 발생하면 교사의 외부 검증 작업 시간이 부담스러움. **v0.6은 외부 검증을 🔴 필수검토 학생에 한정**해 작업 부담을 절반 이하로 줄임.

**비용 효율 (v0.6 기준)**: 학급 30명에서 🔴 필수검토 약 10~15% (3~4명) → 외부 검증 3~4건만.

## 🆕 v0.6 발동 조건 (강화됨)

다음을 모두 만족할 때만 5.5 단계 진입:

1. 의장이 학생을 **🔴 필수검토 라벨**로 분류 (회색지대 2건 이상 또는 만장일치 미충족 후 재판정 변경 등)
2. 한 조건 이상이 외부 검증으로 도움받을 수 있는 회색지대

🟡 권장검토 학생(회색지대 1건)은 **외부 검증으로 보내지 않는다**. Claude 페르소나 토론 결과만으로 요약 산출물을 생성하고, 교사가 최종 검토.

🟢 자동확정 학생은 당연히 외부 검증 X.

## 모델 비종속 설계

이 단계는 **특정 외부 모델을 강제하지 않는다.** 사용자(교사)가 선택:
- ChatGPT (GPT-4, GPT-5)
- Gemini (Google)
- Perplexity
- 또는 다른 LLM

시스템은 **검토 요청 패키지를 생성**하고, 사용자가 그것을 외부 모델에 던지고 **결과를 다시 가져오면** 자동 통합한다.

## Claude 처리 절차

### 단계 1: 🔴 필수검토 학생·조건 자동 식별 (v0.6 변경)

먼저 `03_chairman.yaml`(또는 chairman 단계 직전 yaml)에서 **🔴 필수검토 라벨** 학생을 모두 추출. 그 학생들의 회색지대 항목만 외부 검증 대상.

🟡 권장검토 학생의 회색지대는 외부 검증 대상에서 **제외**. (Claude 페르소나 결과만으로 요약 산출물 생성.)

```yaml
회색지대_목록:
  - 학번: 2-3-2
    이름: 강종민
    조건: 5
    조건_내용: "에세이의 어느 부분이 시의 어느 부분에 반영되었는지..."
    보수_평가자_의견: 미충족 (확신도 60%)
    탐색_평가자_의견: 충족 (확신도 55%)
    
  - 학번: 2-3-3
    이름: 권민기
    조건: 1
    ...
```

### 단계 2: 🆕 v0.8 통합 검토 요청문 생성 (별도 폴더에 모음)

학생 폴더가 아닌 **통합 폴더에 모든 외부 검증 자료를 모음**:

```
outputs/[평가명]/_외부검증/
├── README.md                              작업 안내 (자동 생성)
├── _검토목록.md                            전체 검토 대상 명단 (자동 생성)
├── [학번]_[이름]_조건N.md                  통합 검토 요청문 (단일 파일에 모든 정보)
└── [학번]_[이름]_조건N_결과.md             결과 입력란
```

`prompts/external_model_request.md` v0.8 템플릿을 사용해 **단일 파일에 모든 정보 통합**:
- 과제명·성취기준
- 평가 조건 + 전체 루브릭
- 학생 산출물 전문 (자동 익명화)
- 다른 평가자(보수+탐색) 의견
- 함정 패턴 메모

→ 교사가 이 파일 하나만 복사해 ChatGPT/Gemini에 던지면 됨. **별도 학생자료 첨부 불필요**.

### 단계 3: 사용자에게 안내 메시지

```
[외부 모델 교차 검증 단계]

회색지대 항목 X건이 발견되었습니다:
  · 2-3-2 강종민, 조건 ⑤ (1건)
  · 2-3-3 권민기, 조건 ①, ⑤ (2건)
  · ...

각 회색지대마다 외부 모델 검토 패키지를 다음 위치에 생성했습니다:
  outputs/시창작_해설서/per_student/2-3-2_강종민/external_check/
  outputs/시창작_해설서/per_student/2-3-3_권민기/external_check/
  ...

[선생님께서 하실 일]
1. ChatGPT 또는 Gemini 또는 다른 LLM을 별도로 여세요
2. 각 회색지대 학생의 "조건[N]_검토_요청문.md" 내용을 그대로 복사
3. 외부 모델에 붙여넣고 결과를 받으세요
4. 결과를 같은 폴더의 "조건[N]_결과_입력란.md"에 붙여넣어 주세요
5. 모든 회색지대 검토가 끝나면 알려주세요 (또는 "외부 검토 완료"라고 말씀해 주세요)

⏱️ 예상 시간: 회색지대 1건당 약 5분
💰 비용: 외부 모델 무료 티어로도 가능 (GPT 무료 / Gemini 무료)
```

### 단계 4: 사용자 입력 대기

Claude Code는 사용자가 "외부 검토 완료"라고 말할 때까지 대기.

이 단계는 동기적(synchronous)이고 사용자 차단이 필요하다 — 외부 모델 결과 없이 다음 단계로 가지 않는다.

### 단계 5: 외부 모델 결과 자동 파싱

사용자가 "외부 검토 완료"라고 알리면 각 `결과_입력란.md` 파일을 읽고 파싱.

표준 형식 (외부 모델에게 이 형식으로 답하도록 요청문에 명시):
```yaml
판정: 충족 또는 미충족
확신도: 0~100
근거: (2~4줄)
인용: (학생 자료에서)
다른 평가자가 놓쳤을 수 있는 단서: (있다면)
```

파싱 실패 시 (외부 모델이 다른 형식으로 답한 경우):
- Claude가 자유 텍스트에서 핵심 정보 추출 시도
- 추출 실패 시 사용자에게 재입력 요청

### 단계 6: 세 의견 비교 — 합의 판정

각 회색지대 항목에 대해:

```
Claude 보수 평가자 의견: [충족/미충족]
Claude 탐색 평가자 의견: [충족/미충족]
외부 모델 의견:        [충족/미충족]
```

판정 로직:

| Case | 패턴 | 결과 |
|---|---|---|
| 1 | 셋 다 같은 결론 | 자동 확정 (확신도 매우 높음) |
| 2 | Claude 분열, 외부가 한쪽 명확 | 외부 결정자 — 그쪽으로 합의 |
| 3 | Claude 분열, 외부도 회색지대 | 🔴 **교사 직접 검증 필수** |
| 4 | Claude 합의, 외부가 반대 | 🟡 **교사 검토 권장** (이의제기) |

### 단계 7: 의장에게 결과 전달

`6_chairman.md`로 가는 입력에 다음을 추가:
- 외부 모델 의견
- 세 의견 합의 결과 (Case 1~4)
- 합의 가능 여부

의장은 이 정보까지 종합해 최종 보고서를 만든다.

### 단계 8: 속기록 보강

`transcript_raw.md`에 외부 모델 의견을 추가:

```markdown
## 【4라운드 — 외부 모델 교차 검증】

▸ **외부 모델 (사용자가 ChatGPT-4에 던짐)**

[외부 모델 의견 전문 그대로]

판정: 충족
확신도: 65%
근거: ...

(회의 + 외부 검토)

## 【합의 판정】

- Claude 보수 평가자: 미충족 (60%)
- Claude 탐색 평가자: 충족 (55%)
- 외부 모델: 충족 (65%)
- 합의 결과: Case 2 (외부 결정자) → 충족으로 합의
- 확신도 자연 하락 적용: 평균 60%
```

## 출력

- `outputs/[평가명]/per_student/[학번]_이름/02_5_external.yaml` (외부 모델 결과 + 합의 판정)
- `transcript_raw.md` 에 4라운드 추가

## 주의사항

- **외부 모델은 강제 아님.** 사용자가 외부 검토를 건너뛰고 싶다면 "외부 검토 생략"이라고 말하면 됨. 그 경우 모든 회색지대가 🔴 교사 직접 검증으로 라벨링됨.
- **외부 모델 결과의 형식이 일정하지 않을 수 있음.** Claude가 자유 텍스트에서도 핵심 정보 추출 가능하도록 설계.
- **외부 모델도 같은 편향에 빠질 수 있음.** 합의가 곧 정답을 의미하지 않음. 메타 평가자가 외부 모델 의견의 신뢰도도 점검.
- **외부 모델 의견을 학생 리포트에 직접 인용하지 말 것.** 속기록에만 기록. 학생 리포트는 의장이 종합 후 자기 톤으로 작성.
- **개인 정보 주의**: 외부 모델에 학생 이름을 노출하지 말 것. 검토 요청문은 익명화된 자료만 포함.

## v0.3 가치 요약

- **선별적 다중 모델**: 회색지대만 외부 검증 → 비용 효율
- **진짜 다양성**: 다른 학습 데이터의 의견으로 Claude 페르소나의 본질적 한계 보완
- **단계적 에스컬레이션**: 회색지대 → 외부 모델 → 교사 검증의 명확한 계층
- **교사 부담 분산**: 모든 회색지대가 교사에게 가지 않음 (외부 모델로 일부 해결)
- **모델 비종속**: GPT, Gemini, Perplexity 등 어떤 모델이든 사용 가능

## ⚠️ 한계 인정

이 메커니즘은 **반자동**입니다. 외부 모델 API 자동 호출이 아니라 사용자가 직접 검토 요청문을 외부 모델에 던지고 결과를 가져오는 방식. **풀 자동화는 v1.0의 영역**입니다.

다만 회색지대가 학급당 5~10건 정도라면 1건당 5분 × 5건 = 약 25분의 추가 시간이 발생합니다. **교사가 5명 모두를 직접 채점하는 시간(약 50~75분)의 절반**이므로 충분히 효율적입니다.
