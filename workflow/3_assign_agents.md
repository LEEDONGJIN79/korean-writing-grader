# Workflow 3: 채점단 동적 구성 (Assign Agents) — v0.2

## 목적
파싱된 루브릭의 N개 항목을 분석해, 각 항목에 맞는 **전문가 에이전트의 역할·페르소나·평가 정책**을 결정한다.

이 단계가 **이 시스템의 핵심**이다. 평가 유형을 미리 정의하지 않고 루브릭만 보고 채점단을 동적으로 구성하되, **v0.2부터는 페르소나 라이브러리를 활용해 진짜 평가 철학의 다양성을 확보**한다.

## v0.2 핵심 변경 사항

- ✅ **페르소나 라이브러리 활용** (`prompts/personas/`)
- ✅ **해석형 항목에는 보수+탐색 페르소나 동시 배치** (1라운드부터 다양성 확보)
- ✅ **보수/탐색 톤이 prompt에 명시적으로 박힘**
- ✅ **만장일치 동조 위험을 1라운드 단계에서 미리 방지**

## 입력
- `outputs/[평가명]/_parsed_rubric.yaml` (Workflow 2의 결과)
- 성취기준
- 과제 지시문 (있을 경우)
- 수업·단원 맥락 (있을 경우)
- (선택) 함정 패턴 메모

## Claude 처리 절차

### 1. 항목 성격 분류 (기존과 동일)

각 항목을 **객관형** 또는 **해석형**으로 분류한다.

| 신호어 | 분류 경향 |
|---|---|
| "갖추었는가", "포함되었는가", "언급되었는가" | 객관형 |
| "의도적인가", "구체적인가", "효과적인가" | 해석형 |
| "X개 이상", "OO자 이상" | 객관형 |
| "충분히", "적절히", "독창적으로" | 해석형 |

분류가 애매하면 **해석형으로 분류**한다.

### 2. 역할명 부여

각 항목에 대해 직책명을 부여한다. (기존과 동일)

예시 매핑은 기존과 같음. 단 **해석형 항목은 역할명 뒤에 `(보수)` 또는 `(탐색)` 표기**.

### 3. 🆕 페르소나 선택 (v0.2 핵심)

`prompts/personas/` 에서 항목 성격과 평가 영역에 맞는 페르소나를 선택한다.

#### 페르소나 라이브러리 (현재 6개)

| 페르소나 | 톤 | 평가 철학 |
|---|:---:|---|
| 형식주의자 | 보수적 | 작품 완성도 우선 |
| 시도_옹호자 | 탐색적 | 학생의 시도에 가치 |
| 텍스트_충실파 | 보수적 | 글에 있는 것만 평가 |
| 맥락_통합파 | 탐색적 | 수업·단원 맥락 통합 |
| 의심파 | 보수적 | 충족 주장에 증거 요구 |
| 호의파 | 탐색적 | 학생 의도를 호의적으로 |

#### 페르소나 선택 규칙

**객관형 항목**:
- 페르소나 1명 배치
- **형식주의자** 또는 **텍스트_충실파** (보수 톤) 권장
- 단독 판정으로 종료 (토론 X)

**해석형 항목** ⭐ 핵심:
- 페르소나 **2명 동시 배치** (보수 1명 + 탐색 1명)
- 같은 조건에 두 페르소나가 1라운드부터 의견 제시
- 두 의견 차이가 곧 회색지대 신호
- 만장일치 미충족 시에만 악마의 변호인 추가

#### 영역별 페르소나 권장 조합 (참고)

| 평가 영역 | 페르소나 조합 |
|---|---|
| 시 창작·해설서 | 형식주의자·시도_옹호자·텍스트_충실파·맥락_통합파·의심파·호의파 |
| 논설문·논증 글쓰기 | 형식주의자·의심파·텍스트_충실파·호의파 |
| 자기 성찰·진정성 평가 | 호의파·텍스트_충실파·맥락_통합파 |
| 소설 분석·비평 | 텍스트_충실파·시도_옹호자·의심파 |
| 단답형·조건부 서술형 | 형식주의자·텍스트_충실파·의심파 |

영역이 명확하지 않으면 메타 에이전트가 루브릭 항목을 보고 가장 적절한 조합을 자동 결정.

### 4. 평가 초점 한 문장 작성 (기존과 동일)

각 에이전트가 무엇을 봐야 하는지 한 문장으로 명료히.

### 5. 토론 정책 결정 (v0.2 업데이트)

| 항목 성격 | 1라운드 | 토론 라운드 |
|---|---|---|
| 객관형 | 페르소나 1명 단독 판정 | 없음 |
| 해석형 | **페르소나 2명(보수+탐색) 동시 판정** | 두 의견 차이가 있으면 진행 |
| 해석형 + 만장일치 미충족 | (같음) | 악마의 변호인 추가 발동 |

### 6. 🆕 v0.2 정책 설정 (확대됨)

```yaml
정책:
  # 기존 정책
  관대성_보정자_투입: 해석형 항목이 1개 이상 있으면 "예"
  악마의_변호인_트리거: "미충족_만장일치"
  메타_평가자_가동: 항상 "예"
  성취기준_연동: "예"
  
  # v0.2 신규 정책
  회색지대_디폴트: "예"              # 평가자가 변경 망설일 때 회색지대로
  확신도_자연하락_규칙: "예"          # 입장 변경 시 새 확신도 ≤ 기존 × 0.7
  양방향_자기점검: "예"              # 1라운드 후 찬·반 단서 모두 점검
```

### 7. 함정 패턴 주의 사항 통합 (기존과 동일)

`caveats/` 폴더 또는 사용자 메모에서 관련 항목의 페르소나 prompt에 자동 주입.

### 8. 사용자에게 채점단 구성 보고

```
[채점단 구성 완료 — v0.2]

전문가 에이전트 8명:

객관형 항목 (페르소나 1명씩):
  ① 시 형식 검사관 (형식주의자, 보수)
  ② 창작 의도 확인관 (텍스트 충실파, 보수)
  ④ 시 구성 설명 확인관 (텍스트 충실파, 보수)

해석형 항목 (페르소나 2명씩 - 보수+탐색):
  ③a 시적 표현 평가자 (텍스트 충실파, 보수)
  ③b 시적 표현 평가자 (시도 옹호자, 탐색)
  ⑤a 에세이-시 연결성 평가자 (의심파, 보수)
  ⑤b 에세이-시 연결성 평가자 (호의파, 탐색)

추가 배치:
  · 관대성 보정자 (해석형 항목 의견 제시, 만장일치 시)
  · 악마의 변호인 (만장일치 미충족 시 자동 발동)
  · 메타 평가자 (토론 종료 후 편향 자가 진단)
  · 의장 1명 (최종 판정 및 보고서 생성)

특징:
  · 해석형 항목 ③, ⑤에는 1라운드부터 보수·탐색 두 시각 동시 의견 제시
  · 만장일치 동조 위험을 1라운드에서 미리 차단
  · 입장 변경 시 확신도 자동 하락, 회색지대 디폴트

이 구성으로 채점을 진행해도 될까요?
```

### 9. 동의 후 다음 단계

구성을 `outputs/[평가명]/_agent_assignments.yaml`에 저장하고 `workflow/4_independent_judge.md`로 진행.

## 주의사항

- **항목 수와 에이전트 수는 1대1이 아님.** 객관형은 1명, 해석형은 2명. 5개 조건 중 2개가 해석형이면 7명.
- **객관형과 해석형 비율을 인위적으로 맞추지 말 것.**
- **해석형에 페르소나 2명을 배치할 때 반드시 보수+탐색 조합.** 같은 톤 2명은 동조 위험을 키움.
- **같은 페르소나를 여러 항목에 중복 배치 가능**. 예: 텍스트_충실파가 조건 ②와 ④를 동시에 맡을 수 있음.
- **페르소나 라이브러리에 없는 새 페르소나가 필요하면** `prompts/personas/` 에 새 파일을 만들고 사용자에게 확인 받기.

## v0.2 가치 요약

- **만장일치 동조의 1라운드 단계 차단**: 해석형 항목에 보수+탐색 페르소나가 동시에 있으면 1라운드부터 회색지대 신호 발생 가능
- **악마의 변호인 의존도 감소**: 동조가 1라운드에서 깨지므로 악마의 변호인이 덜 발동되고, 발동 시에도 동조 위험이 낮음
- **명확한 차별성**: 자유 생성 페르소나보다 라이브러리의 명시적 차별성이 더 안정적
