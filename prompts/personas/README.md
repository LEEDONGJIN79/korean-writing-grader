# 페르소나 라이브러리

이 폴더는 채점 에이전트가 입을 수 있는 **평가 철학별 페르소나**들의 라이브러리입니다.

## 왜 라이브러리인가

같은 LLM 모델 안에서도 **명확히 다른 평가 철학**을 prompt에 강하게 박아두면, 같은 학생 답안을 봐도 다른 결론에 도달하는 정도가 커집니다. 자유 생성된 페르소나보다 사전 정의된 페르소나가 더 일관되고 차별성도 분명합니다.

다만 본질적 한계는 남습니다 — 같은 모델·같은 학습 데이터·같은 통념. 이건 v1.0의 다중 모델 통합으로 가야 풀립니다.

## 6개 기본 페르소나

| 페르소나 | 핵심 철학 | 추천 톤 |
|---|---|---|
| **형식주의자** | 작품 완성도가 시도보다 우선 | 보수적 |
| **시도 옹호자** | 학생의 시도 자체에 가치 | 탐색적 |
| **텍스트 충실파** | 작품 외 정보 배제, 글 자체만 본다 | 보수적 |
| **맥락 통합파** | 학생 학습 배경·단원 맥락 고려 | 탐색적 |
| **의심파** | 충족 주장에는 증거를 요구한다 | 보수적 |
| **호의파** | 학생 의도를 최대한 살린다 | 탐색적 |

세 명은 **보수적(낮은 temperature 효과)**, 세 명은 **탐색적(높은 temperature 효과)** — Prompt로 그 효과를 흉내냅니다.

## 사용 방법

메타 에이전트(workflow/3_assign_agents.md)가 루브릭과 과제 성격을 보고 적절한 페르소나 조합을 선택합니다.

### 권장 조합 예시

- **시 창작·해설서**: 형식주의자 + 시도 옹호자 + 텍스트 충실파 + 맥락 통합파 + 의심파
- **논설문**: 형식주의자 + 의심파 + 텍스트 충실파 + 호의파
- **자기 성찰·진정성 평가**: 호의파 + 텍스트 충실파 + 맥락 통합파
- **소설 분석**: 텍스트 충실파(작품 인용 검증) + 시도 옹호자(창의 해석 인정) + 의심파

조합은 루브릭 항목 수와 일치할 필요 없습니다. **각 루브릭 항목마다 페르소나 1~2명**이 배정되며, 같은 페르소나가 여러 항목을 맡을 수도 있습니다.

## 확장

새 페르소나를 추가하려면 이 폴더에 새 파일을 만들고 같은 양식으로 작성하세요. 메타 에이전트가 자동으로 인식합니다.
