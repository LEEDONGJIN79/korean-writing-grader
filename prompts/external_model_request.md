# 외부 모델 검토 요청문 템플릿 — v0.8 (통합본)

## v0.8 변경 핵심

기존(v0.3~v0.7)은 검토 요청문·학생 자료·결과 입력란이 학생별 폴더에 흩어져 있어 외부 모델에 던질 때 여러 파일을 일일이 합쳐야 했음.

v0.8부터는:
1. **모든 정보를 단일 파일에 통합** (과제명·성취기준·루브릭·학생 글·다른 평가자 의견 모두 포함)
2. **별도 폴더 `outputs/[평가명]/_외부검증/`에 모아 저장** — 학생 폴더 탐색 불필요
3. 교사는 그 폴더에서 파일 하나만 복사 → ChatGPT/Gemini에 그대로 붙여넣기 → 답변 받기

## 파일 명명 규칙

```
outputs/[평가명]/_외부검증/
├── README.md                           작업 안내 (자동 생성)
├── _검토목록.md                          전체 검토 대상 한눈에 (자동 생성)
├── [학번]_[이름]_조건N.md                통합 검토 요청문 (Claude 자동 생성)
└── [학번]_[이름]_조건N_결과.md            결과 입력란 (교사가 외부 모델 답변 붙여넣기)
```

예시:
```
outputs/시감상문/_외부검증/
├── README.md
├── _검토목록.md
├── 2-3-2_이서윤_조건②.md          ← 외부 모델에 던질 파일
├── 2-3-2_이서윤_조건②_결과.md      ← 답변 받아 붙여넣기
├── 2-3-5_김진호_조건④.md
├── 2-3-5_김진호_조건④_결과.md
└── ...
```

## Claude가 이 템플릿을 채우는 방법

### 1. 변수 자동 채움
- `{{과제명}}`, `{{성취기준_코드}}`, `{{성취기준_내용}}`: examples/[평가명]/과제정보.md에서
- `{{조건번호}}`, `{{조건_내용}}`: _parsed_rubric.yaml에서
- `{{전체_루브릭_조건들}}`: 다른 조건들도 참고용으로 모두 포함
- `{{점수_산정_표}}`: 충족 개수→점수 매핑
- `{{자료N_제목}}`, `{{자료N_전문}}`: examples/[평가명]/학생자료/에서 학생 글 전문
- `{{보수_판정}}`, `{{탐색_판정}}`: 01_round1.yaml에서

### 2. 학생 익명화
파일명에는 학번·이름이 들어가지만, **검토 요청문 본문에는 "학생"으로만 표기**. 외부 모델에 신원 노출 X.

학생 글 본문에 학번·이름이 있으면 "[학생ID]"로 치환:
```
원문: "글번호 123. 2-3-2 이서윤..."
치환: "글번호 [학번/이름 제거됨]"
```

### 3. 분량 조정
외부 모델 토큰 제한 고려:
- 학생 자료가 너무 길면 핵심 부분만 (조건 평가에 관련된 부분)
- 다만 임의 요약 금지 — 인용은 원문 그대로

### 4. 함정 패턴 자동 주입
평가 영역별 caveats:
- 시 평가 → "너=연인 편향 주의"
- 논설문 → "약한 논거가 의도된 양보일 수 있음"
- 자기 성찰 → "진정성은 외부자가 측정 불가"

caveats가 없는 영역이면 "없음"으로 표시.

## _외부검증/ 폴더의 README.md (자동 생성)

각 평가의 `_외부검증/` 폴더 안에 다음 안내문이 자동 생성됨:

```markdown
# 외부 검증 작업 안내

이 폴더에는 🔴 필수검토 학생의 검토 요청문이 모여 있습니다.

## 작업 절차

1. 이 폴더의 `_검토목록.md`를 먼저 열어 전체 명단 확인
2. 각 학생 파일(예: `[학번]_[이름]_조건N.md`)을 텍스트 에디터로 열기
3. 전체 복사 (⌘+A → ⌘+C 또는 Ctrl+A → Ctrl+C)
4. ChatGPT 또는 Gemini 새 대화창에 붙여넣기 → 전송
5. 답변 전체 복사
6. 같은 학생의 결과 파일(예: `[학번]_[이름]_조건N_결과.md`)에 붙여넣기 → 저장
7. 모든 학생 완료 후 Claude에게 "외부 검토 완료"

## 효율 팁

- ChatGPT·Gemini 둘 다 쓰면 더 정확 (각자 결과를 결과 파일의 다른 섹션에)
- 한 번에 한 학생씩 처리. 두 학생을 한 대화창에서 처리하지 말 것
- 답변이 yaml 형식을 안 지켜도 OK — Claude가 자유 텍스트도 자동 파싱

## 학생당 예상 시간

5~6분 (외부 모델 응답 시간 포함)
```

## 변경 사항 정리 (v0.7 → v0.8)

| 항목 | v0.7 | v0.8 |
|---|---|---|
| 검토 요청문 위치 | 학생 폴더 내부 | **한 폴더에 모음** |
| 파일 개수/학생 | 3개 (요청문·학생자료·결과) | **2개** (통합요청문·결과) |
| 학생 자료 별도 첨부 | 필요 | **요청문에 포함, 첨부 X** |
| 작업 동선 | 학생 폴더 → 파일 합치기 → 던지기 | **폴더 1개에서 파일 1개만 복사** |
| 평균 작업 시간/학생 | 5분 | **3~4분** |
