# 외부 모델 검토 요청문 템플릿

이 템플릿은 회색지대 학생·조건에 대해 외부 모델(GPT, Gemini 등)에 검토를 요청할 때 사용합니다.

작성 시 변수는 `{{변수명}}` 형식. `5_5_external_model_check.md` 가 자동으로 값을 채웁니다.

```markdown

당신은 한국 고등학교 국어 평가 전문가입니다. 한 학생의 답안에 대해 **단 한 가지 평가 조건**에 대한 독립 판정을 부탁드립니다.

이 학생은 다른 평가자들 사이에서 의견이 엇갈린 케이스(회색지대)입니다. 그러나 그들의 의견에 동조하실 필요는 없습니다. **독립적인 시각**으로 평가해 주세요.

## 답변 형식 (반드시 다음 형식으로)

```yaml
판정: 충족 또는 미충족

```

위 형식을 정확히 지켜주세요. 시스템이 자동 파싱합니다.

## 평가 시 유의 사항

1. **루브릭 외의 기준을 추가하지 마세요.** 위 조건 하나만 평가.
2. **학생을 인격적으로 평가하지 마세요.** 글만 봅니다.
3. **확신할 수 없으면 그렇게 말씀해 주세요.** "회색지대"라는 답도 가능합니다.
4. **소재(축구·야구·등산 등)는 평가 요소가 아닙니다.** 루브릭 요소 충족 여부가 핵심.
5. **표면 어휘에 끌려가지 마세요.** "너", "별" 같은 어휘를 자동으로 특정 코드(연애시 등)로 분류하지 말 것.

이 학생에 대한 정직한 독립 판정이 회색지대 해소에 결정적입니다. 감사합니다.

## 작성 시 시스템 처리 사항

### 1. 학생 익명화
- `{{학번}}`, `{{이름}}` 같은 변수는 사용 금지 — 외부 모델에 학생 신원 노출 X
- 학생 답안 본문에 학번·이름이 있으면 "[학생ID]"로 치환

### 2. 학생 자료 분량 조정
- 외부 모델의 토큰 제한 고려
- 학생 자료 전문이 너무 길면 핵심 부분만 포함 (조건 평가에 관련된 부분)
- 다만 임의 요약 금지 — 인용은 원문 그대로

### 3. 함정 패턴 자동 주입
- 평가 영역별 함정 패턴 메모가 있으면 자동 포함
- 시 평가 → "너=연인 편향 주의"
- 논설문 → "약한 논거가 의도된 양보일 수 있음 주의"
- 등

### 4. 답변 형식 강조
- 외부 모델은 형식을 지키지 않을 가능성 있음
- 형식 명시 + 예시 + "시스템이 자동 파싱" 안내 모두 포함
- 그래도 형식 안 맞으면 Claude가 자유 텍스트에서 핵심 정보 추출

### 5. 길이 조정
- 검토 요청문 전체가 너무 길지 않도록 (외부 모델 토큰 절약)
- 한 검토 요청문 = 한 조건만 (여러 조건을 한꺼번에 보내지 말 것)

## 외부 모델별 호환성

이 템플릿은 모델 비종속으로 설계되었지만, 모델별 미세 조정이 가능:

| 모델 | 비고 |
|---|---|
| ChatGPT (GPT-4/5) | 일반적 호환. YAML 형식 잘 지킴 |
| Gemini | 마크다운 헤더 지킴. YAML 가끔 흐트러짐 → 파싱 보강 |
| Perplexity | 검색 기능으로 인해 외부 자료 인용할 수 있음 → 무시 지시 추가 권장 |
| Claude (다른 페르소나) | 같은 모델이라 의미 없음. 사용 X |
