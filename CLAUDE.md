# Korean Writing Grader — Claude Code 진입점

이 폴더는 **국어과 서·논술형 평가를 다중 에이전트 토론·합의로 채점하는 도구**입니다.
이 파일은 Claude Code가 진입 시 자동으로 읽는 안내문입니다.

**현재 버전: v0.8**

---

## 🆕 v0.8 핵심 변경 사항 — 외부 검증 파일 구조 단순화

**v0.7까지의 문제**: 외부 검증 자료가 각 학생 폴더에 흩어져 있어 교사가 일일이 폴더를 돌아다녀야 했음. 또한 검토 요청문과 학생 자료를 따로 외부 모델에 올려야 했음.

**v0.8 해결**:
1. **모든 외부 검증 자료를 단일 폴더에 모음**:
   ```
   outputs/[평가명]/_외부검증/
   ├── README.md                  작업 안내
   ├── _검토목록.md                 전체 명단
   ├── [학번]_[이름]_조건N.md       통합 검토 요청문 (단일 파일에 모든 정보)
   ├── [학번]_[이름]_조건N_결과.md   결과 입력란
   └── ...
   ```

2. **통합 검토 요청문에 모든 정보 포함**:
   - 과제명·성취기준
   - 평가 조건 + 전체 루브릭
   - 학생 산출물 전문 (자동 익명화)
   - 다른 평가자(보수+탐색) 의견
   - 함정 패턴 메모

3. **학생 자료 별도 첨부 불필요** — 검토 요청문 하나만 외부 모델에 던지면 끝

학생당 작업 시간 약 5분 → **3~4분**

⚠️ **`per_student/[학번]_이름/external_check/` 폴더는 v0.8부터 사용하지 않음.** 모든 외부 검증 자료는 통합 폴더에.

## v0.7 누적 메커니즘 — "1명 검증 → 모드 선택 → 자동 완주"

**v0.6까지의 문제**: 학급 채점 시 사용자가 묶음마다 응답을 기다려야 해서 교사가 컴퓨터 앞에 묶임.

**v0.7 해결**:
1. **첫 학생 1명을 먼저 정밀 채점** → 사용자가 결과 확인 후 OK
2. **정석 vs 효율 모드 선택**
3. **효율 모드 선택 시**: 5명 묶음 자동 진행 + 산출물 일괄 생성까지 **사용자 확인 없이 끝까지**

→ **교사가 수업·회의·퇴근 후 돌아왔을 때 모든 결과가 완성**되어 있는 시스템.

### 효율 모드 행동 규칙 (Claude 필수 준수)

효율 모드 진입 시 다음을 반드시 따라야 함:

1. ❌ **묶음 끝났다고 "다음으로 가시겠어요?" 묻지 말 것**
2. ✅ 다음 묶음으로 즉시 자동 진행
3. ✅ 각 묶음 결과는 짧은 요약 한 줄로 (점수·라벨만)
4. ✅ 자세한 내용은 산출물 파일에
5. ✅ 5명을 한 번에 병렬 처리 (Agent 도구 최대 25개 동시 호출)
6. ⚠️ **오류 발생 시에만 멈춤** — 학생 자료 누락, 권한 오류, 응답 형식 깨짐 등
7. ✅ 모든 학생 채점 후 산출물 일괄 생성 (v0.6 차등 정책)
8. ✅ 🔴 필수검토 학생의 **`_외부검증/` 통합 폴더 + 통합 요청문** 자동 생성 (v0.8, 누락 금지)
9. ✅ 최종 보고는 한 번만 (모든 작업 완료 후)

상세는 `workflow/4_independent_judge.md` 참조.

## v0.4 누적 메커니즘 — 학급 단위 운용 흐름

v0.1~v0.3는 학생 1명을 채점하는 워크플로우를 정교화했다면, **v0.4는 학급 전체(20~30명)를 효율적으로 채점하는 운용 방식**을 정리했습니다.

새 운용 5단계:
```
1단계: 채점 완료 (전체 학생 1라운드~토론, 회색지대 큐 누적)
2단계: 잠정 산출물 일괄 생성 (회색지대는 "확인 필요" 표시)
3단계: 회색지대 외부 검증 (교사가 ChatGPT/Gemini에 한꺼번에 던지기)
4단계: 산출물 자동 수정 (외부 검증 결과 반영)
5단계: 완료 (최종 채점표 확정, 교사 검토 권장 학생 안내)
```

상세는 `workflow/_운용흐름.md` 참조.

**핵심 이점**: 외부 검증을 학급 단위로 묶어 교사 시간 부담을 30분 안에 끝낼 수 있음.

## 🎯 이 도구가 하는 일

교사가 다음을 입력하면:
- **과제명** (자유 문자열)
- **루브릭** (한글·Word·PDF·사진 등 자유 형식)
- **성취기준** (교육과정 코드 + 내용)
- **학생 산출물** (1명 또는 다수)

시스템이 자동으로:
1. 루브릭을 파싱해 평가 항목 N개를 추출한다
2. 메타 에이전트가 항목별 전문가 에이전트를 동적으로 구성한다
3. 항목 성격(객관형/해석형)에 따라 토론 방식을 결정한다
4. 다중 에이전트가 독립 판정 → 토론·합의에 도달한다
5. 학생 리포트, 토론 속기록, 채점표 엑셀을 생성한다

## 🏛️ 핵심 설계 원칙

### 원칙 1: 평가 유형 불가지(agnostic)
이 시스템은 시 창작·논설문·소설 분석·자기 성찰 등 어떤 서·논술형 평가에도 작동합니다. **루브릭이 평가를 정의**하며, 시스템은 그 루브릭을 따릅니다. 평가 종류를 미리 분류하지 마세요.

### 원칙 2: AI는 판정하지 않고 의심한다
모든 결과는 "충족/미충족" 이진이 아니라 **확신도(%)**로 표기합니다. 신뢰도가 낮거나 회색 지대인 항목은 자동으로 "교사 확인 필요(?)"로 표시합니다. **최종 판정권은 항상 교사에게** 있습니다.

### 원칙 3: 만장일치는 위험 신호
다섯 에이전트가 미충족으로 만장일치할 경우, **악마의 변호인(Devil's Advocate)** 을 자동 발동해 반대 해석을 시도합니다. 이는 단일 방향 동조를 방지합니다.

### 원칙 4: 다층 해석을 적극 검토하라
해석형 항목에서는 표면적 어휘에 끌려가지 말고, 학생의 의도를 호의적으로 다층적으로 읽으세요. 한국어 통념(예: "너" = 연인)에 기계적으로 분류하지 마세요. **관대성 보정자(Charitable Reader)** 가 이 역할을 전담합니다.

### 원칙 5: 모든 과정은 속기록으로 남긴다
요약본이 아닌 **실제 발언 그대로**의 속기록을 생성합니다. 교사가 AI 판정을 단어 단위로 검증·반박할 수 있어야 합니다. 속기록은 **교사 검토 전용**입니다(학생에게 공개하지 않음).

### 원칙 6: 학생 자료는 외부로 나가지 않는다
모든 처리는 로컬(사용자의 Claude Code 환경)에서 이루어집니다. API 서버에 학생 자료를 업로드하지 마세요.

### ⭐ 0단계: 권한 자가 진단 (필수 사전 점검)

사용자가 다음과 같은 명령을 처음 던졌을 때 Claude는 **반드시 먼저 파일 접근 권한을 자가 진단**해야 한다:

- 설치 요청 ("이 도구를 내 Documents에 설치해줘")
- 채점 시작 요청 ("examples/X 채점해줘")
- 파일·폴더 조작 요청

#### 권한 자가 진단 절차

조용히 다음 세 가지를 시도:
1. `ls ~/Documents` 또는 사용자 홈 폴더 확인
2. `mkdir -p /tmp/kwg_permission_test && ls /tmp/kwg_permission_test`
3. 새 파일 쓰기 테스트 (예: `/tmp/kwg_permission_test/test.txt`에 짧은 내용 쓰기 후 삭제)

#### 권한이 있을 때

사용자에게 짧게 보고하고 본 작업으로 진행:
> "파일 접근 권한 확인됨. 설치를 시작합니다..."

#### 권한이 없을 때

본 작업 중단하고 사용자에게 **운영체제별로 명확한 안내**:

```
파일 접근 권한이 활성화되어 있지 않은 것 같습니다.

[macOS 사용자]
1. Claude Desktop 메뉴 → Claude → Settings... (⌘ + ,)
2. "Developer" 탭 선택
3. "Local Agent Mode" 또는 "Allow Claude to access local files" 옵션을 ON으로 변경
4. 시스템 권한 요청 팝업이 뜨면 "허용" 클릭
5. (필요 시) 시스템 환경설정 → 보안 및 개인 정보 보호 → 전체 디스크 접근 권한에서 Claude를 체크
6. Claude Desktop을 재시작

[Windows 사용자]
1. Claude Desktop 좌측 하단 톱니바퀴(⚙) 아이콘 클릭
2. "Developer" 또는 "고급 설정" 메뉴 진입
3. "Local Agent Mode" 옵션을 ON
4. (필요 시) Claude 아이콘을 우클릭 → "관리자 권한으로 실행"

[학교 컴퓨터인 경우]
보안 정책으로 막혀 있을 수 있습니다. 정보 부서에 문의하시거나
개인 컴퓨터에서 사용해 주세요.

활성화 후 같은 명령을 다시 입력해 주세요.
설치/채점 작업은 권한이 확인되어야 진행할 수 있습니다.
```

⚠️ 권한이 없으면 **본 작업을 진행하지 말 것**. 무리하게 시도하면 중간에 실패해 사용자에게 더 큰 혼란.

### Claude의 표준 처리 순서 (권한 확인 후):

1. **`workflow/1_setup.md`** 를 읽고 입력 정보를 정리한다.
2. **`workflow/2_parse_rubric.md`** 를 따라 루브릭 파일을 읽고 항목을 추출한다. 추출 결과를 사용자에게 한 번 보여주고 확인을 받는다 (실수 방지 장치).
3. **`workflow/3_assign_agents.md`** 를 따라 메타 에이전트로서 항목별 전문가 에이전트를 구성한다. 각 항목을 "객관형"/"해석형"으로 분류한다.
4. **`workflow/4_independent_judge.md`** 를 따라 각 학생에 대해 전문가 에이전트들을 `Agent` 도구로 병렬 호출한다.
   - 🆕 객관형 항목: 페르소나 1명 단독 판정
   - 🆕 해석형 항목: **보수+탐색 페르소나 2명 동시 판정** (다양성 확보)
   - 각 에이전트는 `prompts/specialist_template.md` + `prompts/personas/[페르소나].md` 결합
5. **`workflow/5_debate.md`** 를 따라 토론 라운드를 진행한다:
   - 해석형 항목 보수+탐색 의견 차이 → 자연 회색지대
   - 만장일치 미충족 (보수+탐색 모두 미충족) 시에만 `prompts/devils_advocate.md` 발동
   - 발동 시 평가자들은 **양방향 자기점검** (찬·반 단서 모두 검토)
   - 입장 변경 시 **확신도 자연 하락 규칙** 적용 (≤ 기존 × 0.7)
   - 망설일 때 **회색지대 디폴트**
   - `prompts/meta_evaluator.md` 가 편향 자가 진단

5.5. 🆕 **`workflow/5_5_external_model_check.md`** — 회색지대 학생만 외부 모델 교차 검증:
   - 회색지대 학생·조건을 자동 식별
   - 각 케이스에 대해 `prompts/external_model_request.md` 템플릿으로 **검토 요청문 자동 생성**
   - `outputs/[평가명]/_외부검증/` 통합 폴더에 통합 검토 요청문 저장 (v0.8)
   - 사용자에게 안내: "ChatGPT/Gemini 등에 던지고 결과를 받아 입력란에 붙여넣어 주세요"
   - 사용자가 "외부 검토 완료" 알리면 결과 자동 파싱
   - 회색지대 없으면 이 단계 자동 건너뛰기
6. **`workflow/6_chairman.md`** 를 따라 의장 에이전트로서 최종 결정을 내린다. 
   - 🆕 외부 모델 결과(`02_5_external.yaml`)가 있으면 **세 시각 합의 판정 로직** 적용
   - Case 1·2 (합의 가능): 자동 확정 또는 외부 결정자 방향
   - Case 3 (셋 다 다름): 🔴 **교사 직접 검증 필수**
   - Case 4 (Claude 합의 vs 외부 반대): 🟡 **이의제기 가능** 표시
7. **`workflow/7_outputs.md`** 를 따라 세 가지 산출물을 `outputs/` 폴더에 생성한다:
   - 학생 개인 리포트 (`templates/student_report.md` 기반)
   - 토론 속기록 (`templates/debate_transcript.md` 기반, 교사 검토 전용)
   - 채점표 엑셀 (`templates/scoring_table.xlsx` 기반)

## ⚠️ Claude가 절대 해서는 안 되는 일

1. **점수를 단정하지 말 것** — 항상 확신도와 함께 보고하라
2. **루브릭에 없는 평가 기준을 추가하지 말 것** — 시스템은 교사의 루브릭만 따른다
3. **표면 어휘로 의미를 단정하지 말 것** — "너", "별", "꽃밭" 같은 어휘를 기계적으로 분류하지 마라
4. **만장일치를 신뢰하지 말 것** — 만장일치는 오히려 동조 위험 신호다. 악마의 변호인을 반드시 발동하라
5. **학생 자료를 외부로 전송하지 말 것** — 모든 처리는 로컬에서
6. **속기록을 학생용 리포트에 포함하지 말 것** — 속기록은 교사 검토 전용
7. **사용자에게 묻지 않고 채점을 시작하지 말 것** — 루브릭 파싱 후 확인을 반드시 받아라
8. **권한 없이 본 작업을 강행하지 말 것** — 사용자 명령 시 가장 먼저 파일 접근 권한 자가 진단
9. **🆕 효율 모드 진입 후 묶음마다 사용자 확인 묻지 말 것** — 사용자가 효율 모드를 선택했다는 건 자리를 비울 수 있다는 뜻. 오류 없이 진행되면 끝까지 자동 완주. 산출물 생성까지 한 번에 완료 후 최종 보고만

- **엑셀 생성**: `anthropic-skills:xlsx` 스킬 활용
