## 작성 원칙

### 1. 원문 보존
- 각 에이전트의 `원_발언` 필드를 **있는 그대로** 옮긴다
- 짧게 만들거나 정리하지 말 것
- 어색한 표현, 망설임의 흔적, "제가 생각하기로는..." 같은 발화 마커도 그대로

### 2. 시간 표기
- 첫 발언을 "+0:00"으로 시작
- 발언마다 +N:M 형식으로 상대 시간 표기 (실제 시간 아님, 토론의 흐름을 보여주기 위함)
- AI 호출은 거의 동시에 일어나지만, **토론의 흐름**을 보여주는 의례적 표기

### 3. 라운드 구분
- 【1라운드】, 【2라운드 — 악마의 변호인】, 【3라운드 — 재토론】 같은 헤더를 명확히
- 무토론 케이스(모든 항목 객관형 + 만장일치)는 1라운드와 의장 결정만으로 끝

### 4. 분량
- 학생 1명 1~5쪽
- 토론이 활발할수록 길어짐 (정상)
- 무토론 케이스는 1~2쪽

### 5. 발견된 흥미로운 케이스 표시
- 입장 변경: ⚪ (조용한 표기)
- 동조 의심: ⚠️
- 회색지대: ?
- 다층 해석 시도: 🔍

### 6. 절대 하지 말 것
- 발언을 짧게 줄이거나 요약하지 말 것
- 에이전트의 발언을 "정리해서" 다른 문장으로 바꾸지 말 것
- "결국 충족으로 합의" 같은 메타 정리만 적지 말 것 (반드시 원 발언 인용)

## 교사가 이 속기록을 활용하는 방법

1. **회색 지대·필수 검토 학생**의 속기록을 먼저 읽는다
2. AI가 어디서 망설였는지, 어디서 다층 해석을 했는지 본다
3. AI가 놓친 학생의 의도가 있다면 직접 학생과 면담
4. 최종 점수를 확정한 후 학생 리포트의 점수 표기를 수정 (필요 시)
5. 이의제기가 들어왔을 때 이 속기록을 증거 자료로 활용 가능
