# 학생 리포트 — 자동확정용 (간략)

🟢 **자동확정** 학생용 양식. 만장일치 충족 + 회색지대 없음 + 확신도 평균 80%↑.

리포트 파일은 **생성하지 않음**. 대신 채점표.md/csv의 학생 행에 다음 정보만 기록.

## 채점표 행에 들어갈 정보

```yaml
학번: [학번]
이름: [이름]
점수: [점수] (자동확정 ✓)
한_줄_코멘트: "[학생 글의 가장 인상 깊은 점 1문장]"
   (예: "할아버지의 침묵을 '이불'로 감각화한 발상이 인상 깊었다")
```

## 한 줄 코멘트 작성 원칙

- **30~80자 사이**
- 학생 글에서 가장 빛나는 한 가지만 짚기
- 점수 사유 X (이미 만장일치 충족이므로 불필요)
- 부드러운 종결어미 ("~한 발상이 또렷했다", "~한 점이 좋았다")

## 학생이 점수 사유를 묻는다면

별도로 풀 리포트를 사후 생성할 수 있음:

```
사용자 명령: "[학번] 학생 리포트 자세히 만들어줘"
→ Claude가 그 학생에 한해 풀 리포트 생성 (templates/student_report.md 기반)
```

평소엔 자동확정 학생의 풀 리포트를 미리 만들지 말 것 (시간·토큰 낭비).

## yaml 파일은 보관

`outputs/[평가명]/per_student/[학번]_이름/` 폴더에는 다음만 보관:
- 01_round1.yaml (감사 추적)
- 03_chairman.yaml (최종 판정)

속기록.md, 리포트.md는 **생성하지 않음**. transcript_raw.md도 생략 가능.

## 채점표에서의 표기 예시

```
| 학번 | 이름 | ① | ② | ③ | ④ | ⑤ | 점수 | 라벨 | 한 줄 코멘트 |
|------|------|---|---|---|---|---|------|------|-------------|
| 2-3-1 | 강건우 | ✓ | ✓ | ✓ | ✓ | ✓ | 7 | 🟢 | 침묵을 이불로 감각화한 발상이 또렷 |
```
