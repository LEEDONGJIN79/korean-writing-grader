# 🛠️ 설치 및 실행 가이드 — v1.0

이 문서는 **Korean Writing Grader를 처음 받아 쓰는 교사를 위한 단계별 가이드**입니다. 코딩 경험이 없어도 따라할 수 있도록 자세히 안내합니다.

> **v1.0 — 앵커 매트릭스 제거 + 채점 팀 수 선택**
>
> v0.9에서 도입한 앵커 매트릭스(점수대별 예시 자동 주입)가 소재·분량 편향을 일으켜 분리. 채점은 **루브릭만으로** 판정.
>
> 시작 시 사용자가 **3가지** 선택:
> 1. 채점 교사 수준 (🔵 엄격 / 🟢 느슨)
> 2. 의장 수준 (🔵 엄격 / ⚖️ 균형 / 🟢 느슨)
> 3. 🆕 **채점 팀 수** (1팀 순차 / **5팀 추천** / 10팀 / 전체병렬)
>
> **v0.9 4단계 절차 유지**:
> 1. **채점 교사 1명** (사용자가 엄격/느슨 선택) → 1차 채점
> 2. **학생 페르소나** (모든 학생, 고2 톤) → 이의제기
> 3. **검토 교사 2명** (보수+탐색) → 학생 이의 재심 (이의 있을 때만)
> 4. **의장** (사용자가 엄격/균형/느슨 선택) → 최종 결정
>
> v0.7 효율 모드, v0.8 외부 검증 통합 폴더 모두 유지.

## 📋 사전 준비

### 1. Claude Desktop 설치 (필수)

[Claude Desktop](https://claude.ai/download) 접속 → 자기 운영체제에 맞는 버전 다운로드:

| OS | 설치 방법 |
|---|---|
| **macOS** | `.dmg` 파일 다운로드 → 더블클릭 → Applications 폴더로 드래그 |
| **Windows** | `.exe` 파일 다운로드 → 더블클릭 → 설치 마법사 따라가기 |

### 2. Claude 구독 (필수)

[claude.ai](https://claude.ai)에서 다음 중 하나 구독:
- **Claude Pro** ($20/월): 일반 사용자용, 충분히 사용 가능
- **Claude Max** ($100~200/월): 대규모 학급·자주 사용 시

⚠️ 무료 플랜으로는 사용 횟수가 부족합니다.

### 3. ChatGPT 또는 Gemini 계정 (선택, 권장)

외부 모델 교차 검증 단계에서 사용합니다. 회색지대 학생만 검증하므로 무료 플랜으로도 가능:
- **ChatGPT**: [chatgpt.com](https://chatgpt.com) (Plus 권장, 무료도 가능)
- **Gemini**: [gemini.google.com](https://gemini.google.com) (Google 계정만 있으면 무료)

## 📥 1단계: 저장소 다운로드

### ⭐ 방법 A: Claude에게 직접 요청 (가장 쉬움, 권장)

**압축 풀기·터미널 명령어 같은 거 안 해도 됩니다.** Claude Desktop이 알아서 받아옵니다.

#### 전제 조건
Claude Desktop의 **파일 시스템 접근 권한(Local Agent Mode)** 이 활성화되어 있어야 합니다.

- macOS: Claude Desktop 설정 → "Allow Claude to access files" 또는 "Local Agent" 옵션 활성화
- 처음 실행 시 권한 요청 팝업이 뜨면 **허용** 클릭

#### 명령
Claude Desktop 새 대화창에 다음을 그대로 붙여넣습니다:

```
https://github.com/LEEDONGJIN79/korean-writing-grader
이 도구를 내 Documents 폴더에 설치해줘.
설치 후 폴더 구조를 확인해서 모든 게 준비됐는지 점검해줘.
```

#### Claude가 자동으로 할 일
1. `~/Documents/`(또는 사용자 지정 위치)로 이동
2. `git clone` 실행해 저장소 다운로드
3. 폴더 안의 `CLAUDE.md`를 읽어 시스템 인식
4. 폴더 구조·필수 파일 점검
5. "설치 완료. 첫 평가 정보를 알려주세요" 보고

→ 1~2분 안에 끝. ZIP 해제·이름 변경·이동 같은 작업 일절 없음.

#### 만약 Claude가 "git이 설치되지 않았다"고 답하면

Claude에게 그대로 말하면 됩니다:

```
git이 없으면 ZIP 다운로드 방식으로 받아줘.
```

Claude가 `curl`로 ZIP 다운로드 → `unzip`으로 압축 해제 → 폴더 이름 정리까지 자동 처리합니다.

## 📝 3단계: 첫 평가 만들기

### 폴더 구조 만들기

`korean-writing-grader/examples/` 안에 자기 평가 폴더를 만듭니다. 이름은 자유롭게:

```
examples/
└── 2026_1학기_시감상문/        ← 폴더 이름은 자유, 한글 가능
    ├── 과제정보.md             ← 필수
    ├── 루브릭.png              ← 선택 (한글/Word/PDF/이미지 무엇이든)
    └── 학생자료/                ← 선택 (학생 답안 모음)
        └── ...
```

### 과제정보.md 작성

`examples/_레퍼런스_시창작_해설서/과제정보.md`를 복사해 자기 평가에 맞게 수정합니다.

**최소한의 양식**:

```markdown
# [과제명] — [학교/학년]

## 과제명
[자유 입력]

## 대상
[학년·반]

## 성취기준
[12국어01-03] [성취기준 본문]

## 루브릭

### 평가 조건
① [조건 1]
② [조건 2]
③ [조건 3]
...

### 점수 산정
| 충족 개수 | 점수 |
|---|---|
| 5개 모두 | 7 |
| 4개 | 6 |
...

## 학생 산출물
[어떤 종류의 글을 받는지. 예: 감상문 1편, 800자]

## 함정 패턴 주의 사항 (선택)
[이 평가에서 미리 알아두어야 할 편향이 있다면 적기. 예: "한국어 통념상 '너'를 자동으로 연인으로 분류하지 말 것"]
```

⚠️ **루브릭 파일 형식은 자유롭습니다.** 한글(.hwp), Word(.docx), PDF, 사진(.png/.jpg) 어느 것이든 Claude가 읽을 수 있어요. 다만 사진의 경우 **글자가 또렷이 보이도록** 찍어주세요.

### 학생 자료 배치

`examples/[과제명]/학생자료/` 폴더 안에 학생 답안을 넣습니다. 파일 이름에 학번·이름을 포함하면 좋습니다:

```
학생자료/
├── [2-3-1] 김민준.docx
├── [2-3-2] 이서윤.docx
├── [2-3-3] 박지호.docx
...
```

⚠️ **학생 자료는 `.gitignore`에 의해 GitHub에 절대 업로드되지 않습니다.** 안심하세요.

## 🎯 4단계: 채점 실행

### 자연어로 명령

Claude에게 다음과 같이 말합니다:

```
examples/2026_1학기_시감상문 채점해줘.
과제정보.md에 다 적혀 있어.
```

### 🆕 v0.7 운용 흐름

```
[1단계-A] 첫 학생 1명 정밀 채점
       ↓
   Claude: "2-3-1 김민준, 7점, 🟢 자동확정. 결과 맞으세요?"
       ↓
   교사: "맞아, 계속해줘"
       ↓
[1단계-B] 모드 선택
       ↓
   Claude: "정석 모드(묶음마다 확인) vs 효율 모드(자동 완주) 어느 쪽?"
       ↓
   교사: "효율 모드" ⭐
       ↓
[1단계-C] 효율 모드 자동 완주 — 사용자 자리 비워도 됨
   · 5명씩 묶어 채점 (응답 한 번에 끝까지)
   · 회색지대 학생·조건 자동 식별
       ↓
[2단계] v0.6 차등 산출물 일괄 생성
   🟢 자동확정 (~60%): 채점표 한 줄
   🟡 권장검토 (~25%): 요약 산출물
   🔴 필수검토 (~15%): 풀 산출물 + _외부검증/ 폴더에 검토 요청문
       ↓
[3단계] 🔴 필수검토 학생만 외부 검증 (교사 작업, 15~20분)
   ChatGPT/Gemini에 검토 요청문 던지기
       ↓
[4~5단계] 산출물 수정 + 최종 완료
```

### 5단계 운용 흐름 상세

```
1단계: 채점 완료
       └─ Claude가 학생 1명씩 6개 페르소나로 채점
       └─ 회색지대 학생을 자동 식별
       └─ ⏱️ 학생 1명당 약 1~2분, 30명이면 약 30~60분

2단계: 잠정 산출물 생성
       └─ outputs/[과제명]/ 폴더에 자동 생성
       └─ 학생별 리포트.md, 속기록.md
       └─ 채점표.md, 채점표.csv, _채점요약.md
       └─ 🔴 필수검토 학생은 outputs/[평가명]/_외부검증/ 폴더에 통합 검토 요청문

3단계: 외부 모델 검증 (선생님 작업, 약 30분)
       └─ Claude가 회색지대 명단 안내
       └─ 선생님이 검토 요청문을 ChatGPT/Gemini에 던지기
       └─ 결과를 각 학생의 결과_입력란.md에 붙여넣기

4단계: 산출물 자동 수정
       └─ "외부 검토 완료"라고 알리면 Claude가 자동 처리
       └─ 영향받는 학생들의 산출물 업데이트

5단계: 완료
       └─ 최종 채점표 확정
       └─ 🟢 자동확정 / 🟡 권장검토 / 🔴 필수검토 라벨 안내
```

## 🔍 5단계: 외부 모델 검증 (3단계 상세)

회색지대 학생이 있을 때만 진행합니다.

### Claude가 안내하는 형태

```
[외부 모델 교차 검증 단계]

회색지대 항목 5건이 발견되었습니다:
  · 2-3-2 이서윤, 조건 ② (1건)
  · 2-3-5 김진호, 조건 ④ (1건)
  · 2-3-8 백선우, 조건 ① (1건)
  · ...

각 회색지대마다 외부 모델 검토 패키지를 다음 위치에 생성했습니다:
  outputs/2026_1학기_시감상문/_외부검증/
  ...

선생님께서 하실 일:
1. ChatGPT 또는 Gemini 또는 다른 LLM을 별도로 여세요
2. 각 회색지대 학생의 "조건N_검토_요청문.md" 내용을 그대로 복사
3. 외부 모델에 붙여넣고 결과를 받으세요
4. 결과를 같은 폴더의 "조건N_결과_입력란.md"에 붙여넣어 주세요
5. 모든 회색지대 검토가 끝나면 "외부 검토 완료"라고 말씀해 주세요
```

### 구체적 작업 절차

각 회색지대 학생에 대해 다음을 반복합니다 (학생당 약 5~6분):

1. **요청문 열기**: `outputs/[과제명]/_외부검증/[학번]_[이름]_조건N.md`를 텍스트 에디터나 마크다운 뷰어로 열기 (v0.8: 단일 통합 폴더에 모두 모여 있음)
2. **전체 복사**: Cmd+A → Cmd+C (macOS) / Ctrl+A → Ctrl+C (Windows)
3. **외부 모델에 붙여넣기**: ChatGPT 또는 Gemini 새 대화 창에 붙여넣기 → 전송
4. **답변 받기**: 외부 모델이 yaml 형식으로 답변 (판정/확신도/근거/인용 등)
5. **답변 복사**: 답변 전체 복사
6. **결과 입력란에 붙여넣기**: 같은 폴더의 `조건N_결과_입력란.md` 열고 해당 위치에 붙여넣기
7. **저장**

### 두 모델을 함께 쓰면 더 정확

ChatGPT와 Gemini 둘 다 결과를 받으면 **두 외부 모델 + Claude 페르소나 6명 = 8개 시각**으로 회색지대를 검증할 수 있습니다. 합의 신뢰도가 더 올라가요.

### 모든 검증이 끝나면

Claude에게 다음과 같이 말합니다:

```
외부 검토 완료
```

Claude가 자동으로:
1. 모든 결과_입력란.md를 파싱
2. 영향받는 학생들의 산출물(리포트·속기록·채점표) 자동 수정
3. 최종 검토 권장 학생 명단 안내

## 📂 6단계: 결과 확인

### 어디서 결과를 보나

```
outputs/[과제명]/
├── _채점요약.md                ← 한 페이지 메타 보고서 (먼저 보세요)
├── 채점표.md                   ← 마크다운 표 (보기용)
├── 채점표.csv                  ← 엑셀로 변환할 수 있는 CSV
│
└── per_student/
    ├── 2-3-1_김민준/
    │   ├── 리포트.md           ← 학생에게 줄 수 있는 글
    │   └── 속기록.md           ← 교사 검토 전용 (회의록)
    └── ...
```

### 선생님이 우선 봐야 할 파일

1. **`_채점요약.md`** 먼저 — 학급 전체 그림
2. **`채점표.md`** — 누가 몇 점인지 한눈에
3. **🔴 필수검토** 라벨 학생의 **`속기록.md`** — 직접 면담 권장
4. **🟡 권장검토** 라벨 학생의 **`속기록.md`** — 시간 있으면 검토
5. **🟢 자동확정** 학생은 **`리포트.md`** 만 빠르게 훑기

### NEIS·기록부 입력 흐름

1. `채점표.csv`를 Excel/Numbers로 열기
2. **"교사확정점수"** 컬럼이 비어 있음 → 검토 후 확정 점수 입력
3. 학생용 리포트는 그대로 출력하거나 메시지로 전송
4. **속기록은 학생에게 절대 공개하지 않기** (AI 토론 과정 포함)

## ❓ 자주 묻는 질문 (FAQ)

### Q1. Claude Desktop이 폴더를 못 찾아요
→ Claude Desktop을 완전히 종료(Cmd+Q)했다가 다시 실행해 보세요. 또는 폴더 경로에 특수 문자가 있는지 확인.

### Q2. CLAUDE.md를 안 읽는 것 같아요
→ "이 폴더의 CLAUDE.md 읽어줘" 라고 명시적으로 요청하세요.

### Q3. 한글 파일(.hwp)을 못 읽어요
→ macOS에서 한글 파일 읽기는 다음 중 하나가 필요합니다:
- **한글 뷰어 설치**: 한글 파일 텍스트 추출이 가능해짐
- **변환 후 사용**: 한글 → PDF 또는 Word로 변환 후 업로드

윈도우는 한글이 설치되어 있으면 자동 인식.

### Q4. 학생이 많은데 시간이 너무 오래 걸려요
→ 26명 기준 약 1~2시간 예상. 다음 방법으로 빠르게:
- **묶음 처리 요청**: "한 번에 5명씩 묶어서 처리해줘"
- **객관형 항목만 빠르게**: "객관형 조건은 한 페르소나에게 일괄 판정해줘"
- **회색지대만 정밀**: "충족 합의가 명확한 학생은 빠르게 넘기고, 회색지대만 자세히"

### Q5. 외부 모델 검증을 건너뛰고 싶어요
→ "외부 검증 생략" 이라고 말하면 모든 회색지대가 🔴 교사 직접 검증으로 라벨링되고 5단계로 넘어갑니다.

### Q6. 채점 결과가 마음에 안 들어요
→ 시스템은 보조 도구입니다. 최종 점수는 선생님이 확정하세요. `채점표.csv`의 "교사확정점수" 컬럼에 자유롭게 입력. 다만 다음을 점검해보세요:
- 루브릭이 명확한가?
- 함정 패턴 메모를 빠뜨렸나?
- 회색지대 학생을 직접 면담해 봤나?

### Q7. 다른 평가 영역(논설문·자기소개서 등)에도 쓸 수 있나요?
→ **네.** 시스템은 평가 유형을 미리 정해두지 않습니다. 루브릭만 바꾸면 어떤 서·논술형이든 채점 가능. 시 창작·해설서는 시범 사례일 뿐입니다.

### Q8. 학생 자료가 GitHub에 올라가나요?
→ **아니요.** `.gitignore`로 outputs/, 학생 자료 폴더는 모두 제외됩니다. 외부 모델 검증 시에도 학생 이름·학번은 자동 익명화.

### Q9. Claude가 너무 많은 토큰을 써서 사용량 한도에 걸려요
→ Pro 플랜의 한도가 부족하면 Max 플랜으로 업그레이드하거나, 학급을 묶음으로 나눠 하루에 10~15명씩 처리.

### Q10. 외부 모델 검증 결과 형식이 어긋났어요
→ ChatGPT나 Gemini가 yaml 형식을 잘 지키지 않을 수도 있어요. Claude가 자유 텍스트에서도 핵심 정보를 추출할 수 있지만, 너무 어긋나면 "외부 모델에 다시 던져서 형식 맞춰달라고 해주세요" 안내가 나옵니다.
