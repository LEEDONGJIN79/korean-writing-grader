## 🤔 어떤 도구인가요

학생의 서·논술형 글(시 창작, 논설문, 감상문, 분석문, 자기소개서 등 무엇이든)을 **6명의 AI 전문가 에이전트가 다른 평가 철학으로 채점**하고, **회색지대 학생만 외부 모델(ChatGPT/Gemini)로 교차 검증**한 뒤, **교사가 최종 확정**하는 도구입니다.

### 단일 LLM 채점과 무엇이 다른가

| | 단일 LLM 채점 | Korean Writing Grader |
|---|---|---|
| 채점자 | 1명의 AI | **6명의 페르소나 + 외부 모델** |
| 다양성 | 한 모델의 한 시각 | **보수+탐색 + 외부 학습 데이터** |
| 편향 검증 | 없음 | **만장일치 시 악마의 변호인, 회색지대 시 외부 모델** |
| 결과 표현 | 점수 통보 | **확신도·회색지대 명시, 교사 확인 권장 표시** |
| 검증 가능성 | 블랙박스 | **모든 발언 속기록으로 저장** |
| 최종 권한 | AI 점수 | **항상 교사 확정** |

## 🚀 시작하기

### 1. 사전 준비

- [Claude Desktop](https://claude.ai/download) 설치
- Claude Pro 또는 Max 구독
- (선택) 외부 검증용: ChatGPT 또는 Gemini 계정

### 2. 저장소 다운로드

```bash
git clone https://github.com/LEEDONGJIN79/korean-writing-grader.git
cd korean-writing-grader
```

또는 GitHub 페이지에서 **"Code → Download ZIP"** 으로 받아 압축 풀기.

### 3. Claude Desktop에서 폴더 열기

Claude Desktop을 실행하고 `korean-writing-grader` 폴더를 선택합니다. Claude가 자동으로 `CLAUDE.md`를 읽어 시스템을 인식합니다.

### 4. 자기 평가 만들기

```bash
mkdir -p examples/내_평가_이름
```

`examples/_레퍼런스_시창작_해설서/과제정보.md`를 복사해 자기 평가에 맞게 수정하고, 학생 자료를 `examples/내_평가_이름/학생자료/` 안에 넣으세요.

### 5. Claude에게 자연어로 명령

```
"examples/내_평가_이름 채점해줘. 과제정보.md에 다 적혀 있어."
```

Claude가 5단계 운용 흐름을 자동으로 따라갑니다.

## 📋 입력·출력

### 교사가 입력하는 것 (4종)
1. **과제명** (자유 문자열)
2. **루브릭** (한글·Word·PDF·이미지·텍스트 무엇이든)
3. **성취기준** (교육과정 코드 + 내용)
4. **학생 산출물** (1명 또는 다수)

### 시스템이 생성하는 것 (3종)
1. **학생 개인 리포트** — 학생 공개용 (강점·보완점·성취기준 도달 진단)
2. **전문가 토론 속기록** — 교사 검토 전용 (모든 AI 발언 회의록)
3. **채점표 엑셀** — NEIS 입력용 (선생님 확정 점수 칸 비어있음)

## 🏛️ 핵심 메커니즘 (v0.4)

### 학급 단위 5단계 운용

```
1단계: 채점 완료 (전체 학생 1라운드~토론)
2단계: 잠정 산출물 일괄 생성 (회색지대는 "확인 필요" 표시)
3단계: 회색지대 외부 검증 (교사가 ChatGPT/Gemini에 한꺼번에)
4단계: 산출물 자동 수정 (외부 검증 결과 반영)
5단계: 완료 (최종 채점표 확정)
```

### 6가지 페르소나

루브릭 항목 성격에 따라 메타 에이전트가 자동 선택·배치:

| 페르소나 | 톤 | 한 줄 정체성 |
|---|:---:|---|
| 형식주의자 | 보수 | 작품 완성도가 시도보다 먼저 |
| 시도 옹호자 | 탐색 | 학생의 시도를 먼저 본다 |
| 텍스트 충실파 | 보수 | 글에 있는 것만 평가 |
| 맥락 통합파 | 탐색 | 수업·단원 맥락 안에서 |
| 의심파 | 보수 | 충족 주장에 증거 요구 |
| 호의파 | 탐색 | 학생 의도를 가장 호의적으로 |

### 동조 위험 차단 메커니즘

- **양방향 자기 점검** — "반대 가설"이 아닌 "찬·반 단서 모두 재점검"
- **확신도 자연 하락 규칙** — 입장 변경 시 새 확신도 ≤ 기존 × 0.7
- **회색지대 디폴트** — 망설일 때 변경이 아닌 회색지대로
- **악마의 변호인 발동 제한** — 만장일치 미충족 + 확신도 70%↑일 때만
- **외부 모델 교차 검증** — 회색지대만 ChatGPT/Gemini로 검증

## 🔒 학생 자료 보안

- 모든 처리는 **사용자의 로컬 환경**에서 진행
- 학생 자료는 외부 클라우드로 전송되지 않음
- `outputs/`, 학생 자료 폴더는 모두 `.gitignore`로 Git 추적 제외
- 외부 모델 검증 시 학생 이름·학번은 자동 익명화

## ⚠️ 사용 시 주의사항

1. **시스템이 매긴 점수를 그대로 신뢰하지 마세요.** "교사 확인 필요" 표시된 학생은 반드시 직접 검토.
2. **루브릭이 시스템의 전부입니다.** 명확한 루브릭이 명확한 채점을 만듭니다.
3. **속기록은 학생에게 보여주지 마세요.** AI 토론 과정은 학생에게 혼란을 줄 수 있습니다.
4. **이의제기 대응 시** 속기록과 외부 모델 검증 결과가 가장 강력한 증거 자료입니다.
5. **점수는 교사가 확정**하고 NEIS·기록부에 입력합니다.

## 📚 참고 연구

이 도구는 다음 학술 흐름의 영향을 받았습니다:

- **다중 에이전트 토론** (Multi-Agent Debate): ChatEval (thunlp, 2023), DebateLLM (instadeepai, 2024)
- **악마의 변호인 / 회색지대 검증**: Adaptive Stability Detection (arxiv 2510.12697)
- **AutoSCORE**: 다중 에이전트 기반 자동 채점 (arxiv 2509.21910)
- **KICE 연구**: 한국교육과정평가원 "교과 서·논술형 평가 자동채점을 위한 인공지능 모델 적용 방안 연구(Ⅱ)" (2025)

KICE의 기계학습 기반 접근과 본 도구의 LLM 기반 접근은 **상호 보완 관계**입니다:
- KICE: 대규모 학습 데이터 기반 정확도·검증 표준
- 본 도구: 학습 데이터 없이 교실 현장의 즉시성·다양성에 대응
