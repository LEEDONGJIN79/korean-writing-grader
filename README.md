## 🤔 어떤 도구인가요

학생의 서·논술형 글(시 창작, 논설문, 감상문, 분석문, 자기소개서 등 무엇이든)을 **실제 학교의 평가 절차(채점교사 → 학생 이의 → 검토교사 → 의장)를 그대로 모방한 4단계 직렬 모델로 채점**하고, **의견이 갈리는 학생만 외부 모델(ChatGPT/Gemini)로 교차 검증**한 뒤, **교사가 최종 확정**하는 도구입니다.

### 단일 LLM 채점과 무엇이 다른가

| | 단일 LLM 채점 | Korean Writing Grader |
|---|---|---|
| 채점자 | 1명의 AI | **채점교사 + 학생 + 검토교사 2명 + 의장 (4단계 페르소나) + 외부 모델** |
| 다양성 | 한 모델의 한 시각 | **보수+탐색 + 외부 학습 데이터** |
| 편향 검증 | 없음 | **만장일치 시 악마의 변호인, 회색지대 시 외부 모델** |
| 결과 표현 | 점수 통보 | **확신도·회색지대 명시, 교사 확인 권장 표시** |
| 검증 가능성 | 블랙박스 | **모든 발언 속기록으로 저장** |
| 최종 권한 | AI 점수 | **항상 교사 확정** |

## 🚀 빠른 시작

자세한 단계별 가이드는 **[SETUP.md](./SETUP.md)** 참조.

### 가장 쉬운 3단계 (압축 풀기·터미널 명령 X)

1. **Claude Desktop 설치** ([claude.ai/download](https://claude.ai/download)) + Claude Pro/Max 구독
2. **Claude에게 직접 설치 요청** — 새 대화창에 그대로 붙여넣기:
   ```
   https://github.com/LEEDONGJIN79/korean-writing-grader
   이 도구를 내 Documents 폴더에 설치해줘.
   ```
   → Claude가 자동으로 `git clone`(또는 ZIP 다운로드) + 폴더 인식 + 시스템 점검까지 완료
3. **자연어로 채점 명령**:
   ```
   examples/내_평가 채점해줘
   ```
   → v0.9 4단계 절차: **채점교사 → 학생 이의 → 검토교사 → 의장**
   → 시작 시 **3가지** 선택:
   - 채점교사 수준 (🔵 엄격 / 🟢 느슨)
   - 의장 수준 (🔵 엄격 / ⚖️ 균형 / 🟢 느슨)
   - 🆕 **채점 팀 수** (1팀 순차 / **5팀 추천** / 10팀 / 전체병렬)

### 🆕 v1.0 — 앵커 매트릭스 제거 + 채점 팀 수 사용자 선택

**앵커 매트릭스 제거**: v0.9에서 도입했던 "점수대별 예시" 자동 주입이 실제 채점에서 소재·분량 종속, 앵커 효과 부작용을 드러내 분리. 채점은 **루브릭만으로** 판정.

**채점 팀 수 사용자 선택**: 학급 처리 시간·토큰 비용을 사용자가 직접 선택 가능.

| 옵션 | 학급 25명 시간 | 토큰 비용 | 권장 |
|---|---|---|:---:|
| 1팀 (순차) | 30분~1시간 | 가장 낮음 | 정밀 점검 |
| **5팀** | 5~10분 | 중간 | 일반 수행평가 ⭐ |
| 10팀 | 3~5분 | 중상 | 빠른 확인 |
| 전체병렬 | 2~3분 | 가장 높음 | 시간 부족 |

### 🆕 v0.9 실제 학교 평가 절차 모방

실제 학교의 채점·이의제기·재심·최종 결정 절차를 그대로 옮긴 모델:
- 채점 교사 1명 (사용자가 엄격/느슨 선택)
- 학생 페르소나의 이의제기 (모든 학생, 고2 톤)
- 검토 교사 2명(보수+탐색) 재심 (이의 있을 때만)
- 의장 (사용자가 엄격/균형/느슨 선택) 최종 결정

기존 v0.2~v0.8의 "평가 철학 4~6명 병렬 토론"보다 **더 자연스럽고 학생 시각을 포함**.

### 🆕 v0.7 효율 모드의 가치

```
[효율 모드 사용 시 교사 흐름]
오전 9시: "채점해줘" → 첫 학생 검증 + 효율 모드 선택 (5분)
오전 9:05~12:00: 수업·회의·점심
오후 12:00: 돌아와서 결과 확인 (모든 채점·산출물·외부 검증 패키지 완성)
오후 12:00~12:30: 🔴 필수검토 학생만 ChatGPT/Gemini 검증
오후 12:30~ : NEIS 입력
```

## 📋 입력·출력

### 교사가 입력하는 것 (4종)
1. **과제명** (자유 문자열)
2. **루브릭** (한글·Word·PDF·이미지·텍스트 무엇이든)
3. **성취기준** (교육과정 코드 + 내용)
4. **학생 산출물** (1명 또는 다수)

### 시스템이 생성하는 것 (3종)
1. **학생 개인 리포트** — 학생 공개용 (강점·보완점·성취기준 도달 진단)
2. **전문가 토론 속기록** — 교사 검토 전용 (모든 AI 발언 회의록)
3. **채점표 엑셀** — NEIS 입력용 (선생님 확정 점수 칸 비어있음)

## 🏛️ 핵심 메커니즘 (v1.0)

### v0.9 4단계 직렬 절차

```
① 채점 교사 (사용자 선택: 엄격/느슨)
   → 1차 채점 + 망설임 표시
② 학생 페르소나 (모든 학생, 고2 톤)
   → 점수 동의하거나 이의제기
③ 검토 교사 2명 (보수+탐색, 이의 있는 경우만 병렬)
   → 학생 이의 재심
④ 의장 (사용자 선택: 엄격/균형/느슨)
   → 모든 의견 종합·최종 결정
```

### v0.9 페르소나 8개 라이브러리

`prompts/personas/`:

| 페르소나 | 역할 | 사용자 선택 |
|---|---|:---:|
| 채점교사_엄격 / 채점교사_느슨 | 1차 채점 | ✓ |
| 학생 | 이의제기 (고2 톤) | (전원 자동) |
| 검토교사_보수 / 검토교사_탐색 | 학생 이의 재심 | (이의 시 자동) |
| 의장_엄격 / 의장_균형 / 의장_느슨 | 최종 결정 | ✓ |

기존 v0.2의 6개 평가 철학 페르소나(형식주의자·시도 옹호자 등)는 `prompts/personas/_archive_v02/`에 보존.

### 학급 단위 5단계 운용 (v0.4 유지)

```
1단계: 채점 완료 (모든 학생 4단계 절차)
2단계: 잠정 산출물 일괄 생성 (회색지대는 "확인 필요" 표시)
3단계: 회색지대 외부 검증 (ChatGPT/Gemini에 한꺼번에)
4단계: 산출물 자동 수정 (외부 검증 결과 반영)
5단계: 완료 (최종 채점표 확정)
```

### 동조 위험 차단 메커니즘

- **학생 이의제기 단계** — 모든 학생이 자기 점수에 대해 한 번 발화 (v0.9 신규)
- **검토교사 보수·탐색 분리** — 같은 이의를 두 시각으로 재심
- **외부 모델 교차 검증** — 의견 갈리는 학생만 ChatGPT/Gemini로 검증
- **교사 최종 확정** — AI 점수는 항상 잠정. 교사가 NEIS에 입력

## 🔒 학생 자료 보안

- 모든 처리는 **사용자의 로컬 환경**에서 진행
- 학생 자료는 외부 클라우드로 전송되지 않음
- `outputs/`, 학생 자료 폴더는 모두 `.gitignore`로 Git 추적 제외
- 외부 모델 검증 시 학생 이름·학번은 자동 익명화

## ⚠️ 사용 시 주의사항

1. **시스템이 매긴 점수를 그대로 신뢰하지 마세요.** "교사 확인 필요" 표시된 학생은 반드시 직접 검토.
2. **루브릭이 시스템의 전부입니다.** 명확한 루브릭이 명확한 채점을 만듭니다.
3. **속기록은 학생에게 보여주지 마세요.** AI 토론 과정은 학생에게 혼란을 줄 수 있습니다.
4. **이의제기 대응 시** 속기록과 외부 모델 검증 결과가 가장 강력한 증거 자료입니다.
5. **점수는 교사가 확정**하고 NEIS·기록부에 입력합니다.

## 📚 참고 연구

이 도구는 다음 학술 흐름의 영향을 받았습니다:

- **다중 에이전트 토론** (Multi-Agent Debate): ChatEval (thunlp, 2023), DebateLLM (instadeepai, 2024)
- **악마의 변호인 / 회색지대 검증**: Adaptive Stability Detection (arxiv 2510.12697)
- **AutoSCORE**: 다중 에이전트 기반 자동 채점 (arxiv 2509.21910)
- **KICE 연구**: 한국교육과정평가원 "교과 서·논술형 평가 자동채점을 위한 인공지능 모델 적용 방안 연구(Ⅱ)" (2025)

KICE의 기계학습 기반 접근과 본 도구의 LLM 기반 접근은 **상호 보완 관계**입니다:
- KICE: 대규모 학습 데이터 기반 정확도·검증 표준
- 본 도구: 학습 데이터 없이 교실 현장의 즉시성·다양성에 대응
