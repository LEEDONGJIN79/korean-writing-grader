# 기여 가이드 (Contributing Guide)

Korean Writing Grader에 기여해 주셔서 감사합니다. 이 도구는 **국어 교사 커뮤니티의 공유 문화** 위에서 자라기를 바랍니다.

---

## 🎯 어떤 기여를 환영하는가

### 🌱 새 평가 유형 레퍼런스 추가
- `examples/_레퍼런스_[평가명]/` 폴더로 PR
- 학생 자료는 절대 포함 X (양식·과제 정보·루브릭만)
- 다른 교사가 그 평가 유형에 시스템을 적용할 때 참고용

### 🎭 새 페르소나 제안
- `prompts/personas/[페르소나명].md`
- 기존 6개 페르소나가 잡지 못하는 평가 철학을 가진 경우
- 페르소나 명세 양식은 기존 6개 참조

### 🚨 영역별 함정 패턴 메모
- `caveats/community.yaml` (앞으로 생성 예정)
- "시 평가 시 '너=연인' 편향 주의" 같은 영역 특화 함정
- 시스템이 자동으로 관련 평가에 주입

### 🐛 버그 리포트
- GitHub Issues에 다음을 포함해 등록:
  - Claude Desktop 버전
  - 사용한 명령어
  - 기대한 동작 vs 실제 동작
  - 학생 자료는 절대 첨부 X (익명화된 최소 재현 사례만)

### 📚 사용 후기·개선 제안
- GitHub Discussions에 자유롭게
- 어떤 평가에 적용했는지, 어떤 점이 좋았는지·아쉬웠는지
- 다른 교사들에게 유익한 후기

## ⚠️ 기여 시 절대 지켜야 할 것

### 1. 학생 개인정보 보호 (최우선)
- **학생 자료(글·이름·학번)는 절대 PR/Issues에 포함 X**
- 익명화된 사례 만들 때도 **재식별 가능성** 점검 (학교명·반·특이 경험 등)
- 학생 자료가 우발적으로 포함된 PR은 즉시 reject + 안내

### 2. 라이선스 (CC BY-NC-SA 4.0)
- 기여하시는 코드·문서는 CC BY-NC-SA 4.0으로 공유됩니다
- 다른 라이선스의 코드를 가져올 경우 출처 명시 필수
- 상업적 용도로 변환 시도 금지

### 3. 사용자 친화적 한국어
- 다른 교사가 받아 즉시 이해할 수 있는 표현 사용
- 영어 약어 사용 시 한글 풀이 병기 (예: "NLP(자연어 처리)")
- 코드보다 마크다운 명세가 우선 (Claude Code가 읽어야 하므로)

## 📋 PR 절차

```
1. 저장소 fork
2. 새 브랜치 생성: git checkout -b feature/[기여_내용]
3. 변경 사항 작성
4. 커밋: git commit -m "[type]: 간략한 설명"
5. push: git push origin feature/[기여_내용]
6. GitHub에서 PR 생성
7. PR 본문에 다음 포함:
   - 어떤 변경인지 (1~3줄)
   - 왜 이 변경이 필요한지 (배경)
   - 학생 자료가 포함되지 않았음을 확인
```

### 커밋 메시지 type
- `feat:` 새 기능
- `fix:` 버그 수정
- `docs:` 문서만 변경
- `style:` 포맷·오타 (기능 변화 없음)
- `refactor:` 리팩토링
- `persona:` 새 페르소나 추가
- `caveat:` 함정 패턴 메모 추가
- `example:` 레퍼런스 예시 추가

## 🎨 코드·문서 스타일

### 마크다운
- 한 줄 최대 80자 (가독성)
- 헤더 레벨 일관성 (H1은 파일에 하나만)
- 표로 정리 가능한 정보는 표로

### YAML
- 들여쓰기는 2칸
- 한글 키 사용 권장 (다른 교사 이해 우선)

### 페르소나 작성 시
- 평가 철학 한 줄 정체성 필수
- 빠지기 쉬운 함정 자기 인지
- 발언 예시 1개 이상
- 보수/탐색 톤 명시

## 🗣️ 커뮤니티 규범

- **존중**: 다른 기여자의 관점을 존중하며 토론
- **솔직함**: 시스템의 한계도 솔직히 짚기 (과대 광고 금지)
- **교육적 정직성**: 학생의 학습 성장에 도움이 되는 기여를 우선
- **개인정보 의식**: 항상 학생 자료 보호를 최우선
