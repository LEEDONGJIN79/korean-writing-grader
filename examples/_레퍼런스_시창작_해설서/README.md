# 📚 레퍼런스 예시 — 시 창작 및 자기 해설서

> 이 폴더는 **다른 교사가 자기 평가를 만들 때 참고할 수 있는 레퍼런스 예시**입니다.
> 직접 이 폴더를 수정해 쓰지 마세요. 자기 평가는 `examples/[자기_평가명]/` 폴더를 새로 만들어 진행하세요.

## 무엇에 대한 예시인가

이 폴더는 시스템 개발자(세명고 이동진 교사)가 **개발 초기 시범 사용 시 입력했던 정보**를 그대로 남겨둔 것입니다.

평가 영역: **시 창작 + 자기 해설서**
- 학생이 감정경험 에세이를 먼저 쓰고
- 그 경험을 바탕으로 창작시를 쓴 뒤
- 시에 대한 자기 해설서를 작성하는 3단계 글쓰기 평가

## 폴더 안에 무엇이 있나

```
_레퍼런스_시창작_해설서/
├── README.md          ← 이 파일
└── 과제정보.md         ← 과제명·루브릭·성취기준·함정 패턴 메모 예시
```

`과제정보.md`는 **다른 교사가 자기 평가의 과제정보.md를 만들 때 참고할 수 있는 양식 예시**입니다.

## 자기 평가 만드는 방법

```
examples/[자기_평가명]/
├── 과제정보.md         ← 이 폴더의 과제정보.md를 복사해 자기 평가 정보로 수정
├── 학생자료/           ← (선택) 학생 산출물 폴더
└── 루브릭.png          ← (선택) 원본 루브릭 파일 (한글/Word/PDF/이미지 무엇이든)
```

학생 자료는 **`.gitignore`로 자동 제외**됩니다. GitHub에 학생 개인정보가 올라가지 않으니 안심하세요.

## 시범 결과는 어디 있나

시범 채점 결과(학생 1·2·3번의 리포트·속기록 등)는 `outputs/` 폴더에 생성되었으나, **개인정보 보호를 위해 `.gitignore`로 Git 추적에서 제외**되었습니다.

시스템의 작동 방식과 산출물 양식은 `templates/` 폴더의 양식 명세를 통해 확인할 수 있습니다.
